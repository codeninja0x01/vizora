  <action>
    1. Create `editor/src/middleware.ts`:
       - Import NextResponse and NextRequest from 'next/server'
       - Check for 'better-auth.session_token' cookie (this is Better Auth's default cookie name)
       - If no session cookie AND path starts with '/dashboard': redirect to /login
       - If session cookie exists AND path is /login or /signup: redirect to /dashboard
       - For all other paths: NextResponse.next()
       - Export config.matcher to match: '/dashboard/:path*', '/login', '/signup'
       - IMPORTANT: This is an optimistic redirect only, NOT a security boundary. Real session validation happens in the protected layout server component.

    2. Create `editor/src/app/(protected)/layout.tsx`:
       - Server component (NOT 'use client')
       - Import { auth } from '@/lib/auth'
       - Import { headers } from 'next/headers'
       - Import { redirect } from 'next/navigation'
       - Validate session server-side:
         ```
         const session = await auth.api.getSession({ headers: await headers() })
         if (!session) redirect('/login')
         ```
       - Pass session data to children (can use React context or just render children)
       - Simple layout wrapper: maybe a nav bar at top with user name and sign out button
       - Sign out button calls authClient.signOut() (needs a client component for this)
       - Minimal dashboard nav: link to "Dashboard", link to "API Keys" (/dashboard/api-keys)

    3. Create `editor/src/app/(protected)/dashboard/page.tsx`:
       - Server component
       - Import { auth } from '@/lib/auth'
       - Import { headers } from 'next/headers'
       - Get session and display:
         - Welcome message with user name
         - User email
         - Active organization (if set)
         - Quick links: "Manage API Keys" -> /dashboard/api-keys
       - This is a simple landing page - more dashboard content comes in future phases

    Note: The protected layout handles auth validation. The middleware only provides fast redirects. Even if middleware is bypassed, the layout server component will catch unauthenticated requests.
