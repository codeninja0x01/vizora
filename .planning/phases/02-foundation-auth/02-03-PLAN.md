  <name>Task 1: Create API key generation, hashing, and validation utilities</name>
  <files>
    editor/src/lib/api-keys.ts
  </files>
  <action>
    Create `editor/src/lib/api-keys.ts` with three core functions:

    1. **generateApiKey()**: Returns `{ key: string; hash: string; prefix: string }`
       - Generate key: `sk_live_${crypto.randomBytes(32).toString('base64url')}`
       - The `sk_live_` prefix identifies it as an OpenVideo live API key
       - Compute hash: `crypto.createHash('sha256').update(key).digest('hex')`
       - Extract prefix: first 12 characters of key (e.g., "sk_live_Ab3x") for display
       - Import crypto from 'node:crypto'

    2. **hashApiKey(key: string)**: Returns `string`
       - Simple SHA-256 hash: `crypto.createHash('sha256').update(key).digest('hex')`
       - Used by validateApiKey to hash incoming key for lookup

    3. **validateApiKey(request: Request)**: Returns `Promise<{ userId: string; organizationId: string; tier: string } | null>`
       - Extract Authorization header, check for 'Bearer ' prefix
       - If missing or malformed: return null
       - Hash the provided key with hashApiKey()
       - Query: `prisma.apiKey.findUnique({ where: { keyHash: hash } })`
       - If not found: return null
       - If found but revokedAt is not null: return null (key was revoked)
       - Update lastUsedAt: `prisma.apiKey.update({ where: { id }, data: { lastUsedAt: new Date() } })`
         - Use non-blocking update (don't await, fire-and-forget with .catch() for logging)
       - Return `{ userId: apiKey.userId, organizationId: apiKey.organizationId, tier: apiKey.tier }`
       - Import prisma from './db'

    Type exports:
    - `type ApiKeyContext = { userId: string; organizationId: string; tier: string }`
    - `type GeneratedApiKey = { key: string; hash: string; prefix: string }`
  </action>
  <verify>
    TypeScript compiles without errors.
    Manual verification: generateApiKey() returns object with key starting with "sk_live_", hash is 64-char hex string, prefix is first 12 chars of key.
    hashApiKey(key) produces same hash as generateApiKey().hash for the same key.
  </verify>
  <done>
    API key utility module exports generateApiKey, hashApiKey, and validateApiKey. Keys use sk_live_ prefix, SHA-256 hashing, and validation checks both existence and revocation status. lastUsedAt is updated on each validation (fire-and-forget).
  </done>

    1. Create `editor/src/app/(protected)/dashboard/api-keys/actions.ts`:
       - Mark as 'use server' at top
       - Import { auth } from '@/lib/auth', { prisma } from '@/lib/db', { generateApiKey } from '@/lib/api-keys'
       - Import { headers } from 'next/headers', { revalidatePath } from 'next/cache'

       **createApiKey(name: string)**: Returns `{ key: string } | { error: string }`
       - Get session: `const session = await auth.api.getSession({ headers: await headers() })`
       - If no session: throw Error('Unauthorized')
       - Get active organization from session (session.session.activeOrganizationId)
       - If no active org: return { error: 'No active organization. Please create or select an organization first.' }
       - Generate key: `const { key, hash, prefix } = generateApiKey()`
       - Create in DB: `prisma.apiKey.create({ data: { name, keyHash: hash, keyPrefix: prefix, userId: session.user.id, organizationId: activeOrgId } })`
       - revalidatePath('/dashboard/api-keys')
       - Return `{ key }` - this is the ONLY time the full key is returned

       **revokeApiKey(keyId: string)**: Returns `{ success: boolean } | { error: string }`
       - Get session, verify auth
       - Find key: `prisma.apiKey.findUnique({ where: { id: keyId } })`
       - Verify ownership: key.userId === session.user.id OR user is admin/owner of the key's organization
       - If not found or not authorized: return { error: 'Not found' }
       - Update: `prisma.apiKey.update({ where: { id: keyId }, data: { revokedAt: new Date() } })`
       - revalidatePath('/dashboard/api-keys')
       - Return { success: true }

    2. Create `editor/src/app/(protected)/dashboard/api-keys/page.tsx`:
       - Server component for the key list, with client components for interactive parts
       - Server part:
         - Get session via auth.api.getSession()
         - Get active org from session
         - Query API keys: `prisma.apiKey.findMany({ where: { organizationId: activeOrgId, revokedAt: null }, select: { id: true, name: true, keyPrefix: true, createdAt: true, lastUsedAt: true }, orderBy: { createdAt: 'desc' } })`
         - Render page title "API Keys", description text, create button, and keys table

       - Display table/list with columns:
         - Name
         - Key (show prefix + "..." e.g., "sk_live_Ab3x...")
         - Created (relative date or formatted date)
         - Last Used (relative date or "Never")
         - Actions: Revoke button (with confirmation)

       - Create API Key flow (client component):
         - Button "Create API Key" opens a dialog/modal
         - Dialog has name input field + Create button
         - On create: call createApiKey server action
         - On success: show the generated key in a read-only input with copy button
         - Show warning: "Make sure to copy your API key now. You won't be able to see it again."
         - Close dialog: key is no longer visible

       - Revoke flow (client component):
         - Revoke button shows confirmation: "Are you sure? This cannot be undone."
         - On confirm: call revokeApiKey server action
         - Key disappears from list (revalidation)

       - Empty state: "No API keys yet. Create one to get started."
       - If no active organization: show message to create/select an organization first

       Styling: Follow existing dark theme design system. Use existing UI components (Button, Dialog from Radix) where available. Clean table with subtle borders.
