  <action>
    Present the verification checklist to the user. This is a human-verify checkpoint - no code changes needed. The complete authentication infrastructure was built in Plans 01-04:
    - Email/password signup with email verification (via Resend)
    - GitHub and Google OAuth login
    - Session persistence across browser refresh
    - Protected dashboard with server-side session validation
    - API key generation, listing, and revocation
    - Tiered rate limiting on API endpoints
    - Health check API endpoint demonstrating the full auth + rate limit pipeline
  </action>
  <verify>
    Start the dev server: `cd editor && pnpm dev`

    **1. Signup Flow (AUTH-01)**
    - Visit http://localhost:3000/signup
    - Fill in name, email, password and submit
    - Check email inbox for verification link (if Resend is configured)
    - Click verification link, confirm redirect to login

    **2. Login Flow (AUTH-02)**
    - Visit http://localhost:3000/login
    - Log in with email/password, confirm redirect to /dashboard
    - Verify user name and email displayed on dashboard

    **3. OAuth Login (AUTH-02)**
    - Log out, visit /login
    - Click "Sign in with GitHub" (if OAuth configured)
    - Complete OAuth flow, confirm redirect to /dashboard

    **4. Session Persistence (AUTH-03)**
    - While logged in, refresh browser (F5), confirm still logged in
    - Close and reopen tab, navigate to /dashboard, confirm still logged in

    **5. Route Protection**
    - In incognito window, navigate to /dashboard, confirm redirect to /login

    **6. API Key Generation (AUTH-04)**
    - Navigate to /dashboard/api-keys, click "Create API Key"
    - Enter name, confirm full key is displayed with copy button
    - Close dialog, confirm key in list shows prefix only (sk_live_...)

    **7. API Key Authentication (AUTH-04)**
    - `curl -H "Authorization: Bearer YOUR_KEY" http://localhost:3000/api/v1/health` -> 200
    - `curl http://localhost:3000/api/v1/health` -> 401

    **8. API Key Revocation (AUTH-05)**
    - Revoke key from dashboard
    - `curl -H "Authorization: Bearer REVOKED_KEY" http://localhost:3000/api/v1/health` -> 401

    **9. Rate Limiting (AUTH-06, AUTH-07)**
    - Create new key, run: `for i in $(seq 1 15); do curl -s -o /dev/null -w "%{http_code} " -H "Authorization: Bearer KEY" http://localhost:3000/api/v1/health; done`
    - Confirm first 10 return 200, rest return 429
    - Verify Retry-After header on 429 responses
  </verify>
