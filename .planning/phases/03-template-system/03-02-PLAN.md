  <name>Task 1: Create template CRUD server actions</name>
  <files>editor/src/app/(protected)/dashboard/templates/actions.ts</files>
  <action>
Create editor/src/app/(protected)/dashboard/templates/actions.ts with 'use server' directive.

Follow the same patterns established in editor/src/app/(protected)/dashboard/api-keys/actions.ts:
- Get session via auth.api.getSession({ headers: await headers() })
- Check session exists, throw if not
- Use activeOrganizationId from session for multi-tenant scoping

Implement these server actions:

1. `createTemplate(input: { name: string; description?: string; thumbnailUrl?: string; projectData: Record<string, unknown>; mergeFields: MergeField[]; category?: TemplateCategory; tags?: string[] })`:
   - Validate session
   - Call generateMergeSchema(input.mergeFields) from template-schema.ts to produce mergeSchema
   - Create template via prisma.template.create with userId, organizationId, all input fields, and generated mergeSchema
   - Call revalidatePath('/dashboard/templates')
   - Return created template

2. `updateTemplate(id: string, input: { name?: string; description?: string; thumbnailUrl?: string; projectData?: Record<string, unknown>; mergeFields?: MergeField[]; category?: TemplateCategory; tags?: string[] })`:
   - Validate session
   - Verify template belongs to user: prisma.template.findFirst({ where: { id, userId } })
   - If mergeFields changed, regenerate mergeSchema via generateMergeSchema
   - Update via prisma.template.update
   - Call revalidatePath('/dashboard/templates')
   - Return updated template

3. `deleteTemplate(id: string)`:
   - Validate session
   - Verify ownership (userId match)
   - Delete via prisma.template.delete({ where: { id } })
   - Call revalidatePath('/dashboard/templates')

4. `getTemplates()`:
   - Validate session
   - Query prisma.template.findMany({ where: { userId }, orderBy: { updatedAt: 'desc' } })
   - Return templates array

5. `getTemplateById(id: string)`:
   - Query prisma.template.findUnique({ where: { id } })
   - If template is not public AND userId doesn't match session user, throw 'Not found'
   - Return template or null

6. `getGalleryTemplates(filters?: { category?: string; tags?: string[]; search?: string })`:
   - Query prisma.template.findMany with:
     - where: { isPublic: true }
     - Optional category filter
     - Optional tags filter using hasSome
     - Optional search on name (contains, mode: 'insensitive')
     - orderBy: [{ featured: 'desc' }, { createdAt: 'desc' }]
     - take: 50
   - Return templates array

Import auth from '@/lib/auth', prisma from '@/lib/db', headers from 'next/headers', revalidatePath from 'next/cache'. Import MergeField, TemplateCategory from '@/types/template'. Import generateMergeSchema from '@/lib/template-schema'.

<task type="auto">
  <name>Task 2: Create merge field extraction and application utilities</name>
  <files>editor/src/lib/merge-fields.ts</files>
  <action>
Create editor/src/lib/merge-fields.ts with these functions:

1. `getMergeableProperties(clipType: string): string[]` - Returns the list of properties that can be marked as merge fields for each clip type. Based on the ClipJSON interfaces in json-serialization.ts:
   - 'Text': ['text', 'style.fontSize', 'style.fontFamily', 'style.color', 'left', 'top', 'width', 'height', 'opacity']
   - 'Image': ['src', 'left', 'top', 'width', 'height', 'opacity']
   - 'Video': ['src', 'volume', 'left', 'top', 'width', 'height', 'opacity']
   - 'Audio': ['src', 'volume']
   - 'Caption': ['text', 'style.fontSize', 'style.fontFamily', 'style.color']
   - Others: empty array (Effect, Transition, Placeholder are not mergeable)

2. `extractMergeFields(projectData: Record<string, unknown>, markedFields: Array<{ elementId: string; property: string }>): MergeField[]`:
   - Takes the full ProjectJSON (as Record<string,unknown>) and an array of { elementId, property } pairs indicating which fields the user marked
   - Finds each clip in projectData.clips by matching clip.id === elementId
   - For each marked field, extracts: the element's type, the property value as defaultValue
   - Uses inferFieldType from template-schema.ts to determine fieldType
   - Generates key as `${clip.id}_${property}` (per user decision: field keys use element/property names directly)
   - Returns array of MergeField objects

3. `applyMergeData(projectData: Record<string, unknown>, mergeFields: MergeField[], mergeData: Record<string, unknown>): Record<string, unknown>`:
   - Deep clones projectData using structuredClone()
   - For each mergeField, if mergeData has a value for field.key:
     - Find the clip in cloned projectData.clips by field.elementId
     - Set the property value: if property contains '.', split and traverse (e.g., 'style.fontSize' sets clip.style.fontSize)
     - If mergeData doesn't have the field, use field.defaultValue
   - Returns the modified projectData clone (original untouched)

4. `getPropertyValue(clip: Record<string, unknown>, property: string): unknown`:
   - Handles dot-notation property paths (e.g., 'style.fontSize')
   - Returns the value at the path, or undefined if not found

5. `setPropertyValue(clip: Record<string, unknown>, property: string, value: unknown): void`:
   - Handles dot-notation property paths
   - Sets the value at the path, creating intermediate objects if needed

Import MergeField from '@/types/template'. Import inferFieldType from '@/lib/template-schema'. Use structuredClone() (native, no library needed) for deep cloning.

IMPORTANT: Do NOT use lodash's set/get. Use the custom getPropertyValue/setPropertyValue functions to avoid adding lodash as a dependency for just path access. The dot-notation paths are simple (max 2 levels: 'style.fontSize') so custom implementation is straightforward.
