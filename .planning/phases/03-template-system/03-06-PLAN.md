**1. Create editor/src/app/api/v1/templates/[id]/route.ts** - GET endpoint:

Follow the withApiAuth pattern from editor/src/app/api/v1/health/route.ts:
```typescript
import { withApiAuth } from '@/lib/api-middleware';

export const GET = withApiAuth(async (req, context) => {
  // context has userId, organizationId, tier from API key validation
  const { id } = await req.params; // or extract from URL
  // ... fetch template
});
```

Actually, check the exact pattern in the health route. The withApiAuth middleware wraps the handler and provides auth context.

Implementation:
- Extract template ID from the URL path
- Fetch template: prisma.template.findUnique({ where: { id } })
- If not found: return 404 JSON response
- If template is not public AND organizationId doesn't match auth context: return 403
- Return template as JSON (id, name, description, mergeFields, mergeSchema, category, tags)
- Do NOT return projectData in the API response (it's large and internal)
- Include mergeSchema so API consumers know the validation rules

**2. Create editor/src/app/api/v1/templates/validate/route.ts** - POST endpoint:

Protected by withApiAuth middleware. Validates merge data against a template's stored schema.

Request body:
```json
{
  "templateId": "string",
  "mergeData": { "field_key": "value", ... }
}
```

Implementation:
- Parse request body
- Validate templateId and mergeData are present (return 400 if missing)
- Fetch template: prisma.template.findUnique({ where: { id: templateId } })
- If not found: return 404
- If template is not public AND org doesn't match: return 403
- Call validateMergeData(template.mergeFields as MergeField[], mergeData) from template-schema.ts
- If valid: return 200 with { valid: true, data: validatedData }
- If invalid: return 422 with { valid: false, errors: flattenedErrors }
  - Include field-level error details so API consumers can show specific error messages
  - Format: { valid: false, errors: { fieldErrors: { "field_key": ["error message"] } } }

Per success criteria: "Template merge fields are validated via JSON schema and invalid data returns clear error."

Both endpoints follow the established pattern:
- Use withApiAuth for API key auth + rate limiting
- Return consistent JSON response format
- Include appropriate HTTP status codes
- Rate limiting applied via the middleware (same tiers as health endpoint)
