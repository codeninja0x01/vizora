**1. Create editor/src/components/merge-field-form.tsx** - Client component:

Dynamic form generated from merge field definitions. Props:
- mergeFields: MergeField[] (the template's merge field definitions)
- onChange: (mergeData: Record&lt;string, unknown&gt;) => void (callback when form data changes)

Implementation:
- Initialize form state with defaultValues from each mergeField
- For each merge field, render an appropriate input based on fieldType:
  - 'text': text Input component
  - 'url': text Input with URL placeholder
  - 'number': number Input with step="any"
  - 'color': color input (type="color") with text input showing hex value
- Label each field with a readable name derived from the key (split on '_', take last part, capitalize: "text_abc_fontSize" -> "Font Size")
- On any input change, call onChange with the full form state object
- Style: space-y-4 with Label + Input pairs. Follow dark theme (bg-input, border-input, text-foreground)

Per user decision: "Preview uses original project data as default values -- no placeholder/lorem ipsum generation needed."

**2. Create editor/src/components/template-preview-player.tsx** - Client component:

Playable video preview using the Studio class from openvideo. Props:
- projectData: Record&lt;string, unknown&gt; (ProjectJSON to render)
- width?: number (default 640)
- height?: number (default 360)

Implementation:
- Create a canvas ref and a container div
- On mount, initialize a Studio instance with the canvas element and dimensions from projectData.settings (width/height) or props
- Call studio.loadFromJSON(projectData as ProjectJSON) to load the project data
- Provide simple playback controls: play/pause button, seek slider, current time display
- On unmount, call studio.destroy() to clean up WebGL resources
- Handle loading state (show spinner while Studio initializes and loads clips)
- Handle errors gracefully (show error message if clips fail to load, e.g., missing media URLs)

Note: The preview may not fully render video/image clips if their source URLs are external and not accessible. This is expected -- the preview shows the template structure and text/layout. Full rendering with all media requires the rendering pipeline (Phase 4). The preview IS still playable via Studio's playback mechanism.

Per user decision: "Preview is a playable video, not a static frame -- both in the editor and in the gallery."
