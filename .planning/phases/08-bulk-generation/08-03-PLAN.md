1. Extend the `RenderEvent` interface to include batch event types:
   ```typescript
   export interface RenderEvent {
     type: 'progress' | 'completed' | 'failed' | 'batch.progress' | 'batch.completed';
     renderId: string;
     data: {
       progress?: number;
       outputUrl?: string;
       errorCategory?: string;

       };
     };
   ```

   ```typescript
   const renderBatchMap = new Map<string, string>(); // renderId -> batchId
   ```

3. Add a `registerBatchRender(renderId, userId, batchId)` export that populates both renderOwnerMap and renderBatchMap.

4. In the `completed` event handler, after routing the per-render event:
   - Check if the render has a batchId via renderBatchMap
   - If yes, import and call `getBatchProgress(batchId)` and `updateBatchStatus(batchId)` from batch/tracker
   - Emit a `batch.progress` event to the same user with the aggregate progress
   - If batch is now in a terminal state (completed, partial_failure, failed), also emit `batch.completed`
   - Clean up renderBatchMap entry

5. In the `failed` event handler, apply the same batch logic:
   - Check renderBatchMap for batchId
   - Compute and emit batch.progress
   - Check for terminal state and emit batch.completed if done
   - Clean up renderBatchMap entry

**Important:** Use dynamic `import()` for batch/tracker to avoid circular dependency issues (render-events is loaded early). Wrap batch event emission in try/catch to prevent batch tracking failures from breaking individual render events.

**GET handler (wrapped in withApiAuth):**

1. Extract batchId from URL params (await params promise per Next.js 15+ pattern)
2. Fetch batch with organization scope:
   ```typescript
   const batch = await prisma.batch.findFirst({
     where: { id: batchId, organizationId: context.organizationId },
     include: { template: { select: { name: true } } },
   });
   ```
3. Return 404 if not found
4. If batch status is not 'completed' and not 'partial_failure', return 400: `{ error: 'Batch not ready for download' }`

5. Fetch all completed renders for this batch:
   ```typescript
   const renders = await prisma.render.findMany({
     where: { batchId, status: 'completed', outputUrl: { not: null } },
     select: { id: true, outputUrl: true, batchIndex: true },
     orderBy: { batchIndex: 'asc' },
   });
   ```

6. If no completed renders, return 404: `{ error: 'No completed renders available' }`

7. Create streaming ZIP using archiver:
   ```typescript
   import archiver from 'archiver';

   const archive = archiver('zip', { zlib: { level: 6 } });
   ```

8. Build a manifest tracking skipped renders:
   - Query total render count for batch
   - Track which renders have valid outputUrls and which are missing/expired

9. Create a ReadableStream that bridges archiver output to Web API Response:
   ```typescript
   const stream = new ReadableStream({
     start(controller) {
       archive.on('data', (chunk: Buffer) => controller.enqueue(new Uint8Array(chunk)));
       archive.on('end', () => controller.close());
       archive.on('error', (err) => controller.error(err));

       // Add files to archive
       (async () => {
         for (const render of renders) {
           try {
             const response = await fetch(render.outputUrl!);
             if (!response.ok || !response.body) continue;

             // Use batchIndex for ordered filenames
             const filename = `${String(render.batchIndex ?? 0).padStart(4, '0')}-${render.id}.mp4`;
             archive.append(Buffer.from(await response.arrayBuffer()), { name: filename });
           } catch {
             // Skip failed fetches - record in manifest
             skipped.push(render.id);
           }
         }

         // Add manifest.txt listing any skipped renders
         if (skipped.length > 0) {
           const manifest = `Batch: ${batchId}\nTemplate: ${batch.template.name}\nTotal: ${batch.totalCount}\nIncluded: ${renders.length - skipped.length}\nSkipped: ${skipped.length}\n\nSkipped renders (expired or unavailable):\n${skipped.map(id => `- ${id}`).join('\n')}`;
           archive.append(manifest, { name: 'manifest.txt' });
         }

         archive.finalize();
       })();
     }
   });
   ```

10. Return streaming response:
    ```typescript
    return new Response(stream, {
      headers: {
        'Content-Type': 'application/zip',
        'Content-Disposition': `attachment; filename="batch-${batchId}.zip"`,
        'Cache-Control': 'no-cache',
      },
    });
    ```

**Important considerations:**
- Do NOT use `await archive.finalize()` - let it stream (per research pitfall #4)
- Use `response.arrayBuffer()` per file to stream into archive (for Next.js API routes, full buffer per file is acceptable since individual videos are typically < 50MB)
- For very large batches (>100 renders), this may take time - set appropriate timeout
- Include `export const maxDuration = 300;` for Vercel 5-minute function timeout
