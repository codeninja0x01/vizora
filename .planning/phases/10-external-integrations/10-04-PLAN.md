**base.json:**

```json
{
  "baseUrl": "https://app.openvideo.dev/api/v1",

    "Authorization": "Bearer {{connection.apiKey}}"
  }
}
```

**connections/api-key.json:**
API key connection type with verify endpoint:
```json
{
  "label": "OpenVideo",
  "type": "apikey",
  "parameters": [
    {
      "name": "apiKey",
      "type": "text",
      "label": "API Key",
      "required": true,
      "help": "Find your API key in the OpenVideo dashboard under API Keys. Keys start with sk_live_.",
      "sanitize": true
    }
  ],
  "common": [
    {
      "name": "baseUrl",
      "type": "url",
      "label": "API Base URL",
      "default": "https://app.openvideo.dev/api/v1",
      "advanced": true,
      "help": "Only change this for self-hosted OpenVideo instances."
    }
  ],
  "verify": {
    "url": "{{common.baseUrl}}/me",
    "method": "GET",
    "headers": {
      "Authorization": "Bearer {{parameters.apiKey}}"
    },
    "condition": "{{statusCode}} === 200",
    "error": "Invalid API key. Please check your key in the OpenVideo dashboard."
  }
}
```

The verify section confirms the API key works by calling GET /me. The connection label showing org name + tier is handled by Make's response display pattern.

**modules/renderVideo.json:**
Action module for submitting a video render:

- label: "Render Video"
- description: "Create a new video render from a template with merge data."
- connection: "api-key"

Interface (input fields):
1. templateId: type "select", required, options loaded via RPC "rpc://listTemplatesRpc"
2. mergeData: type "collection", spec loaded dynamically via RPC "rpc://loadMergeFields?templateId={{parameters.templateId}}"

Communication (API request):
```json
{
  "url": "/renders",
  "method": "POST",
  "body": {
    "templateId": "{{parameters.templateId}}",
    "mergeData": "{{parameters.mergeData}}"
  },
  "response": {
    "output": "{{body}}"
  }
}
```

Output (response mapping):
- id: type "text", label "Render ID"
- status: type "text", label "Status"
- templateId: type "text", label "Template ID"
- createdAt: type "date", label "Created At"

**triggers/renderCompleted.json:**
Webhook trigger for render completion events:
- type_id: 10 (instant trigger / webhook)
- label: "Render Completed"
- description: "Triggers when a video render finishes successfully."
- connection: "api-key"

Webhook lifecycle (attach/detach):
```json
{
  "attach": {
    "url": "/webhooks",
    "method": "POST",
    "body": {
      "url": "{{webhook.url}}",
      "enabled": true
    },
    "response": {
      "uid": "{{body.id}}"
    }
  },
  "detach": {
    "url": "/webhooks/{{webhook.uid}}",
    "method": "DELETE"
  },
  "respond": {
    "status": 200,
    "type": "json",
    "body": {
      "accepted": true
    }
  }
}
```

The attach section subscribes by POSTing to /webhooks with Make's webhook URL. The uid (webhook ID) is stored and used in detach to DELETE the subscription. The respond section defines how Make acknowledges incoming webhook events.

Output (what the trigger provides to subsequent modules):
- renderId: type "text", label "Render ID"
- statusUrl: type "url", label "Status URL"

Data mapping (extract from incoming webhook payload):
```json
{
  "output": [
    {
      "name": "renderId",
      "type": "text",
      "label": "Render ID"
    },
    {
      "name": "statusUrl",
      "type": "url",
      "label": "Status URL"
    }
  ],
  "mappable": {
    "renderId": "{{body.data.renderId}}",
    "statusUrl": "{{baseUrl}}/renders/{{body.data.renderId}}"
  }
}
```

Per user decision: minimal payload -- render ID and status URL only.

**rpcs/listTemplatesRpc.json:**
Dynamic options RPC for template dropdown:
```json
{
  "url": "/templates",
  "method": "GET",
  "response": {
    "iterate": "{{body.templates}}",
    "output": {
      "label": "{{item.name}}",
      "value": "{{item.id}}"
    }
  }
}
```

This RPC is referenced by renderVideo.json's templateId field as "rpc://listTemplatesRpc". It fetches all templates and maps them to label/value pairs for the dropdown.

**rpcs/loadMergeFields.json:**
Dynamic fields RPC that loads merge fields for a selected template:
```json
{
  "url": "/templates/{{parameters.templateId}}",
  "method": "GET",
  "response": {
    "iterate": "{{body.mergeFields}}",
    "output": {
      "name": "{{item.key}}",
      "label": "{{item.label}}",
      "type": "{{ifCondition(item.type, 'number', 'number', 'text')}}",
      "required": "{{item.required}}",
      "default": "{{item.defaultValue}}"
    }
  }
}
```

This RPC is referenced by renderVideo.json's mergeData field as "rpc://loadMergeFields?templateId={{parameters.templateId}}". After the user selects a template, Make fetches that template's merge fields and renders them as form inputs.

Type mapping per user decision (merge field types to Make input types):
- text -> "text"
- number -> "number"
- color -> "text" (Make has no native color picker, use text with help text "Enter color value, e.g. #FF0000")
- image -> "url" (image URLs)

Note on Make's template expressions: The exact syntax for conditionals and data transformation may vary. Make uses IML (Integromat Markup Language) for template expressions. If `ifCondition` is not a valid IML function, use an alternative like `{{if(item.type == 'number', 'number', 'text')}}` or simply default all fields to "text" type for simplicity (users can enter any value as text).
