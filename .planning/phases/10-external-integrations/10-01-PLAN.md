<task type="auto">
  <name>Task 2: Create GET /api/v1/templates list endpoint</name>
  <files>editor/src/app/api/v1/templates/route.ts</files>
  <action>
Create GET /api/v1/templates endpoint that returns templates accessible to the authenticated user, with merge field metadata for dynamic dropdown selection in integrations.

Follow the existing withApiAuth and template access control patterns from templates/[id]/route.ts.

Implementation:
1. Wrap handler with withApiAuth
2. Query templates the user can access:
   ```
   prisma.template.findMany({
     where: {
       OR: [
         { organizationId: context.organizationId },
         { isPublic: true },
       ],
     },
     select: {
       id: true,
       name: true,
       description: true,
       category: true,
       thumbnailUrl: true,
       mergeFields: true,
       isPublic: true,
       createdAt: true,
       updatedAt: true,
       // Explicitly exclude projectData (too large, internal only per Phase 3 decision)
     },
     orderBy: { name: 'asc' },
   })
   ```
3. Map results to response format with ISO date strings
4. Return 200 with JSON body: { templates: [...] }

Each template in the response includes mergeFields array so integrations can build dynamic input forms after template selection. The mergeFields are stored as JSONB and each entry has: { key, type, label, required, defaultValue }.

Per user decision: "Template dropdown shows org-owned templates plus cloned gallery templates" -- the OR clause handles this since cloned templates become org-owned.

Do NOT add pagination for this endpoint -- integration platforms load all templates at once for dropdown display. The template count per org will be manageable (tens, not thousands).

Do NOT exclude mergeSchema -- include it so integrations can optionally use it for client-side validation.
  </action>
  <verify>
Verify the file exports GET handler wrapped with withApiAuth. Check that the response includes templates array with mergeFields for each template. Confirm projectData is NOT in the select clause.
  </verify>
  <done>GET /api/v1/templates returns accessible templates with merge field definitions, 401 for invalid key</done>
</task>
