- Implements INodeType from n8n-workflow

**Render Video operation (render:create):**
- templateId field: type 'options' with loadOptionsMethod 'getTemplates' (dynamic dropdown)
- mergeData field: type 'fixedCollection' with multipleValues: true, containing key-value pairs
  - Each entry has: fieldName (string), fieldValue (string)
  - Use displayOptions.show to only appear when templateId is selected
- In execute(): POST /renders with { templateId, mergeData: constructed from fixedCollection }
- Return the render response { id, status, templateId, createdAt }

For dynamic merge fields per template: use loadOptions method 'getTemplateFields' that fetches GET /templates/{templateId} and returns mergeFields as options. The mergeData fixedCollection shows available field names from the selected template.

ALTERNATIVE simpler approach (preferred for v0.1.0): Instead of fixedCollection, use a single JSON field for mergeData where users paste/construct their merge data object. This is simpler, avoids complex n8n dynamic field wiring issues, and power users (n8n target audience) are comfortable with JSON. Add a displayName hint: "Enter merge data as JSON object, e.g. { \"headline\": \"Hello World\" }".

Actually, implement BOTH approaches: a 'mode' selector with options 'Fields' and 'JSON':
- Fields mode: show templateId dropdown + individual merge field inputs loaded dynamically
- JSON mode: show templateId dropdown + single JSON textarea for mergeData

**Get Render Status operation (render:get):**
- renderId field: type 'string', required
- In execute(): GET /renders/{renderId}
- Return the full render response

**List Templates operation (template:getAll):**
- No input fields required
- In execute(): GET /templates
- Return array of templates

**loadOptions methods:**
- getTemplates(): GET /templates, return as INodePropertyOptions[] with { name: t.name, value: t.id }

All HTTP calls use this.helpers.httpRequest() with:

Actually, since credentials have authenticate() that adds the Bearer header, use this.helpers.httpRequestWithAuthentication('openVideoApi', options) pattern instead. This automatically applies credential headers.

**nodes/OpenVideo/OpenVideoTrigger.node.ts:**
- Implements INodeType from n8n-workflow with type: 'trigger'
- displayName: 'OpenVideo Trigger', name: 'openVideoTrigger'
- Description: 'Triggers when a video render completes successfully'
- icon: 'file:openvideo.svg'
- Credentials: openVideoApi

Webhook lifecycle methods (webhookMethods.default):
- checkExists(): GET /webhooks, check if any webhook URL matches the current n8n webhook URL. Return true if found.
- create(): POST /webhooks with { url: webhookUrl, enabled: true }. Store the returned webhook ID in workflow static data (this.getWorkflowStaticData('node')) for later deletion.
- delete(): DELETE /webhooks/{storedId} using the ID stored during create(). Clean up static data.

The n8n webhook URL is obtained via: `this.getNodeWebhookUrl('default')`

webhook() method (processes incoming webhook events):
- Receives the webhook request body (render completion event from OpenVideo)
- Extract render ID and construct status URL
- Return minimal payload per user decision: { renderId: body.data.renderId, statusUrl: `${baseUrl}/renders/${body.data.renderId}` }
- Return format: { workflowData: [this.helpers.returnJsonArray(payload)] }
- The webhook should accept the event and return 200 (n8n handles this automatically)

**Webhook properties:**
```typescript
webhooks: [
  {
    name: 'default',
    httpMethod: 'POST',
    responseMode: 'onReceived',
    path: 'webhook',
  },
],
```

**Error handling in lifecycle:**
- create(): If POST /webhooks fails with 429 (webhook limit), throw clear error: "Webhook limit reached for your organization. Delete unused webhooks in the OpenVideo dashboard."
- delete(): Wrap in try/catch, log errors but don't throw (cleanup should be best-effort)
- checkExists(): If GET /webhooks fails, return false (safe default)
