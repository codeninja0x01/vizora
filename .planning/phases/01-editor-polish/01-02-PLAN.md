  <action>
Refactor `editor/src/components/editor/media-panel/tabbar.tsx` from a horizontal scrollable tab bar to a vertical icon-only sidebar rail (VS Code activity bar pattern):

1. **Layout change**: Convert from horizontal flex (`flex items-center`) to vertical flex (`flex flex-col items-center`).
   - Fixed width: `w-12` (48px) — narrow icon strip.
   - Full height: `h-full`.
   - Background: Use `bg-[oklch(0.16_0_0)]` or similar token from the design system — slightly different shade from the panel content area to create subtle visual separation without borders.
   - Vertical padding: `py-3`, gap between icons: `gap-1`.

2. **Icon buttons**: Each tab renders as a 40x40px icon button (`w-10 h-10`).
   - Icons at 20px (`w-5 h-5`) using the existing Tabler icons from store.ts.
   - Active state: `bg-accent-purple-500/15 text-accent-purple-500` (purple highlight per locked decision).
   - Inactive state: `text-muted-foreground`.
   - Hover state: `hover:bg-white/5 hover:text-foreground`.
   - Transition: `transition-all duration-150`.
   - Keep existing Tooltip wrapping (side="right" instead of side="bottom", sideOffset=8).

3. **Toggle behavior**: Update to support open/close — clicking the active tab collapses the panel content.
   - In store.ts: Add `isPanelOpen` boolean state and `togglePanel` action.
   - When user clicks the already-active tab, toggle `isPanelOpen` to false (collapse content).
   - When user clicks a different tab, set that tab as active AND set `isPanelOpen` to true.
   - Export `isPanelOpen` from the store for MediaPanel to consume.

4. **Remove horizontal scroll logic**: Delete `scrollRef`, `showLeftFade`, `showRightFade`, `checkScrollPosition`, and the fade gradient divs. The vertical layout doesn't need horizontal scrolling.

5. **Remove the `bg-primary/7` background and `px-2 py-2` padding** from the outer container. Replace with the vertical layout styles described above.

6. **Rename the component** from `TabBar` to `ActivityBar` (update the export name). Update the import in `media-panel/index.tsx` accordingly.
  </action>

Update the editor layout to achieve seamless panel flow with no visible borders or gaps:

**editor.tsx changes:**

1. **Remove spacing between panels**: Change the outer container from `space-y-1.5` to `space-y-0` (or remove entirely). Remove `px-2 pb-2` padding — panels should extend edge-to-edge.

2. **Remove ResizableHandle visible styling**: The ResizableHandles currently use `bg-transparent w-1.5` or `!h-1.5`. Keep the handle functional but make it truly invisible:
   - Horizontal handles: `w-0.5 bg-transparent hover:bg-accent-purple-500/30 transition-colors` — invisible by default, subtle purple highlight on hover/drag.
   - Vertical handles: `h-0.5 bg-transparent hover:bg-accent-purple-500/30 transition-colors`.

3. **Panel background differentiation** (per locked decision — no visible borders, background shade differences only):
   - Left panel (MediaPanel): `bg-[var(--panel-background)]` (oklch 0.205).
   - Center area (canvas + timeline): `bg-background` (oklch 0.145, darkest — canvas is the content focus).
   - Right panel (Assistant/Copilot): `bg-[var(--panel-background)]` (same as left).
   - Remove `bg-card` from ResizablePanel classNames — replace with specific panel background colors.

4. **Header integration**: Remove any gap between header and content area. Header should flow into the content without visible separation.

**media-panel/index.tsx changes:**

1. **Layout structure**: The MediaPanel should render as a horizontal flex container:
   - `<div className="flex h-full">` wrapping ActivityBar + content area.
   - ActivityBar on the left (fixed 48px width).
   - Panel content area on the right (flex-1, only visible when isPanelOpen is true).

2. **Conditional content rendering**: Read `isPanelOpen` from `useMediaPanelStore()`.
   - When `isPanelOpen` is true: show the panel content area with the active tab's component.
   - When `isPanelOpen` is false: only the ActivityBar is visible (48px narrow column).

3. **Properties panel integration**: The properties panel currently shows when clips are selected. Keep this behavior but render it WITHIN the content area (replacing the media tab content), not as a separate section. The ActivityBar remains visible regardless.

4. **Remove the Separator** component between TabBar and content. The seamless design uses background shading, not separators.

5. **Content area background**: Use a slightly different shade than the ActivityBar for subtle differentiation: `bg-[var(--panel-background)]` for content, slightly darker shade for ActivityBar.
