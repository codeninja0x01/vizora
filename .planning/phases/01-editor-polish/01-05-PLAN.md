  <action>
Read audio.ts FIRST to understand the current audio clip rendering.

Implement waveform visualization inside audio timeline clips (locked decision: "Audio clips: Always show waveform visualization inside clips"):

1. **Waveform data extraction**:
   - The audio clip likely has access to audio buffer data or a source URL.
   - Check if the clip object has a `peaks`, `waveform`, or `audioBuffer` property (or if the parent clip class from openvideo provides this).
   - If waveform peaks are available: use them directly.
   - If only the audio source URL is available: decode the audio to extract peaks. Use `AudioContext.decodeAudioData()` to get the buffer, then extract peaks by sampling the buffer data at regular intervals.
   - Cache the peaks data on the clip instance to avoid re-computation on every render.
   - Target ~100-200 peaks per visible clip width for smooth waveform at any zoom level.

2. **Waveform rendering** (Pixi.js Graphics):
   - Use the clip's Pixi.js Graphics object to draw the waveform.
   - Color: Use `CLIP_COLORS.audio` (green) for the waveform fill, with slight alpha (0.7-0.8) so the clip background shows through.
   - Alternatively: background is slightly darker green, waveform bars/shape are lighter green.
   - Draw method: For each peak, draw a vertical bar centered vertically in the clip:
     ```
     // For each peak at index i:
     const x = i * (clipWidth / peakCount);
     const barHeight = peak * clipHeight * 0.8; // 80% max height
     const y = (clipHeight - barHeight) / 2;
     graphics.rect(x, y, barWidth, barHeight);
     ```
   - Or use `graphics.moveTo/lineTo` to draw a smooth waveform path (top half mirrored on bottom).
   - The mirrored waveform (top + bottom) looks more professional than bars.

3. **Performance considerations**:
   - Only generate peaks when the clip is first created or when its source changes.
   - Store peaks as a Float32Array on the clip instance.
   - On zoom change: resample peaks to match the new visible width (downsample if zoomed out, use more peaks if zoomed in).
   - Use `beginFill` with alpha for the waveform area, not individual `drawRect` calls for each sample.
   - If peak extraction is async (audio decoding), show the clip with just the green background until peaks are ready, then draw the waveform.

4. **Clip label**: Show the audio clip name overlaid on top of the waveform, positioned at the left edge with slight padding: `text-xs text-white` style, with a subtle background pill/shadow for readability over the waveform.

5. **Fallback**: If waveform data cannot be generated (e.g., cross-origin audio, decode failure), render the audio clip as a solid green background with the clip name. Do not crash or show errors.
