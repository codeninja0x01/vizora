  <name>Task 1: Invoke /frontend-design skill and overhaul globals.css design tokens</name>
  <files>editor/src/app/globals.css</files>
  <action>
CRITICAL FIRST STEP: Before writing any code, invoke the /frontend-design skill to establish the concrete design direction. Provide it with:
- Primary reference: CapCut/Descript visual style (clean, approachable, modern consumer creative tool)
- Mood: Sleek and minimal with generous spacing, content-forward, fewer visible controls
- Accent color: Vibrant purple/violet
- Background: Softer dark (#242424 range)
- Progressive disclosure for complexity
- Web-native feel (not faux-desktop)
- OpenVideo's own distinctive visual identity (not a clone)

Capture the design direction output. Then implement the design token system:

Overhaul `editor/src/app/globals.css` completely. Replace the current green-ish accent tokens with a purple/violet system. Keep the existing Tailwind v4 @theme inline structure but update ALL color values:

1. **Purple accent scale** (in @theme inline):
   - Define `--color-accent-purple-50` through `--color-accent-purple-900` using OKLCH with hue ~300
   - Primary purple at oklch(0.65 0.25 300) range — vibrant but not neon
   - Ensure text-on-purple passes WCAG AA contrast (4.5:1 minimum)

2. **Dark theme backgrounds** (in .dark selector):
   - `--background`: oklch(0.145 0 0) — softest dark, main canvas area (#242424 equivalent)
   - `--card`: oklch(0.18-0.19 0 0) — slightly lighter for card/panel areas
   - `--popover`: oklch(0.17-0.18 0 0) — popover/dropdown background
   - `--muted`: oklch(0.20-0.22 0 0) — muted area background
   - `--secondary`: oklch(0.25-0.28 0 0) — secondary surface
   - NO pure black (#000) anywhere

3. **Panel background shades** for seamless layout:
   - `--panel-background`: oklch(0.205 0 0) — left/right panel shade
   - `--panel-accent`: oklch(0.21 0 0) — subtle panel highlight
   - Add new `--panel-darker` and `--panel-canvas` variables for panel shade differentiation

4. **Semantic accent colors** — update ALL accent/primary/ring to purple:
   - `--primary`: purple-based oklch(0.55-0.65 0.20-0.25 300)
   - `--primary-foreground`: oklch(1 0 0) (white on purple)
   - `--accent`: purple-tinted oklch(0.25 0.03 300)
   - `--accent-foreground`: oklch(0.97 0 0)
   - `--ring`: match primary purple
   - `--sidebar-primary`: match primary purple

5. **Clip type color tokens** (add new tokens in @theme inline):
   - `--color-clip-video`: oklch(0.60 0.20 250) — blue
   - `--color-clip-audio`: oklch(0.65 0.18 145) — green
   - `--color-clip-text`: oklch(0.75 0.15 80) — yellow/amber
   - `--color-clip-effect`: oklch(0.60 0.22 300) — purple
   - `--color-clip-caption`: oklch(0.70 0.15 55) — orange
   - `--color-clip-transition`: oklch(0.55 0.15 200) — cyan

6. **Text hierarchy**:
   - `--foreground`: oklch(0.97 0 0) — primary text (near-white)
   - `--muted-foreground`: oklch(0.60-0.65 0 0) — secondary text
   - Ensure minimum 7:1 contrast for primary text, 4.5:1 for secondary

7. **Animation easings** (add to @theme):
   - `--ease-subtle`: cubic-bezier(0.4, 0, 0.2, 1)
   - `--ease-snappy`: cubic-bezier(0.2, 0, 0, 1)

8. Keep the light mode `:root` values as-is (not in scope for this phase).
9. Keep the existing @utility rules (scrollbar-hidden, scrollbar-thin, etc.) unchanged.
10. Keep `@import "tailwindcss"` and `@import "tw-animate-css"` at top.

Use the /frontend-design skill's output values where they differ from the research defaults, but ensure all locked decisions are honored exactly.
