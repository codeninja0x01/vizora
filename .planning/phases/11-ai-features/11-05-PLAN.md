  <action>
    Create stock footage provider abstraction and style presets:

    1. `editor/src/lib/ai/providers/stock/types.ts` - Stock provider interface:
       - Export interface `StockClip { id: string; url: string; previewUrl: string; thumbnailUrl: string; duration: number; width: number; height: number; tags: string[]; provider: 'pexels' | 'pixabay'; attribution?: string }`
       - Export interface `StockSearchOptions { query: string; perPage?: number; orientation?: 'landscape' | 'portrait' | 'square'; minDuration?: number }`
       - Export interface `IStockProvider { search(options: StockSearchOptions): Promise<StockClip[]>; getProvider(): 'pexels' | 'pixabay' }`

    2. `editor/src/lib/ai/providers/stock/pexels.ts` - Pexels adapter:
       - Class `PexelsProvider implements IStockProvider`
       - Constructor takes `{ apiKey: string }`
       - `search()`: Fetch `GET https://api.pexels.com/videos/search?query={query}&per_page={perPage}&orientation={orientation}` with Authorization header
       - Map response: `data.videos.map(v => ({ id: String(v.id), url: v.video_files.find(f => f.quality === 'hd')?.link || v.video_files[0].link, previewUrl: v.image, thumbnailUrl: v.image, duration: v.duration, width: v.width, height: v.height, tags: [], provider: 'pexels', attribution: 'Video by ' + v.user.name + ' from Pexels' }))`
       - Handle API errors gracefully (return empty array on failure, log error)

    3. `editor/src/lib/ai/providers/stock/pixabay.ts` - Pixabay adapter:
       - Class `PixabayProvider implements IStockProvider`
       - Constructor takes `{ apiKey: string }`
       - `search()`: Fetch `GET https://pixabay.com/api/videos/?key={apiKey}&q={query}&per_page={perPage}`
       - Map response: `data.hits.map(v => ({ id: String(v.id), url: v.videos.large?.url || v.videos.medium?.url, previewUrl: v.videos.tiny?.url, thumbnailUrl: v.videos.tiny?.thumbnail, duration: v.duration, width: v.videos.large?.width || 1920, height: v.videos.large?.height || 1080, tags: v.tags.split(', '), provider: 'pixabay' }))`

    4. `editor/src/lib/ai/providers/stock/factory.ts` - Stock provider factory:
       - Export `StockProviderFactory` with:
         - `static create(provider: 'pexels' | 'pixabay'): IStockProvider`
         - `static getAvailableProviders(): Array<'pexels' | 'pixabay'>` - returns providers with configured API keys
         - `static async searchAll(options: StockSearchOptions): Promise<StockClip[]>` - searches all available providers in parallel, deduplicates by URL, returns combined results sorted by relevance (Pexels first, then Pixabay)

    5. `editor/src/lib/ai/presets/video-style-presets.ts` - Style preset definitions:
       - Export interface `VideoStylePreset { id: string; name: string; description: string; transitionType: string; transitionDuration: number; textStyle: { fontFamily: string; fontSize: number; color: string; fontWeight: string; background?: string; stroke?: string }; pacing: 'slow' | 'medium' | 'fast'; defaultSceneDuration: number }`
       - Export `VIDEO_STYLE_PRESETS: VideoStylePreset[]` with 6 presets:
         1. **corporate**: Inter font, white text, subtle background, fade transitions (1s), medium pacing, 5s scenes
         2. **social-media**: Poppins Bold, white text with black stroke, wipe transitions (0.5s), fast pacing, 3s scenes
         3. **cinematic**: Playfair Display, white text, dissolve transitions (1.5s), slow pacing, 7s scenes
         4. **tutorial**: Roboto, dark text on light background, cut transitions (0.2s), medium pacing, 6s scenes
         5. **energetic**: Montserrat ExtraBold, vibrant yellow text, slide transitions (0.3s), fast pacing, 2.5s scenes
         6. **elegant**: Lora italic, light text, crossfade transitions (1.2s), slow pacing, 6s scenes
       - Export `getVideoStyleById(id: string): VideoStylePreset`

  <action>
    Create the text-to-video assembly service and API:

    1. `editor/src/lib/ai/services/text-to-video-service.ts` - Assembly service:
       - Install `@anthropic-ai/sdk`: `pnpm add @anthropic-ai/sdk --filter editor`
       - Export class `TextToVideoService`
       - Constructor takes `{ anthropicApiKey: string }`
       - Method `generateFromStoryboard(scenes: Scene[], styleId: string): Promise<ProjectComposition>`:
         - Step 1: Call `generateSearchQueries(scenes)` to get stock footage keywords per scene
         - Step 2: For each scene, call `StockProviderFactory.searchAll({ query, perPage: 5 })` in parallel
         - Step 3: Pick best clip per scene (first result, filtering for minimum duration >= scene duration)
         - Step 4: Call `assembleComposition(scenes, clips, style)` to build project
         - Return ProjectComposition
       - Private `generateSearchQueries(scenes: Scene[]): Promise<string[]>`:
         - Use Anthropic SDK: `anthropic.messages.create({ model: 'claude-sonnet-4-5', max_tokens: 1024, messages: [{ role: 'user', content: prompt }] })`
         - Prompt: "For each scene description, generate 2-3 stock footage search keywords optimized for Pexels/Pixabay. Return JSON array of strings, one query per scene. Scenes: {scenes.map(s => s.description).join('; ')}"
         - Parse JSON response, validate it's an array matching scene count
         - Fallback: if parsing fails, extract keywords directly from scene descriptions (split on spaces, take nouns/adjectives)
       - Private `assembleComposition(scenes, clips, style): ProjectComposition`:
         - Build a project JSON structure compatible with the editor's ProjectJSON format
         - For each scene/clip pair, create a video element:
           - `type: 'Video'`, `src: clip.url`, `display: { from: startTimeUs, to: endTimeUs }` (microseconds)
           - `width: 1920, height: 1080` (or from clip dimensions, scaled to fit)
           - Calculate startTime cumulatively from scene durations
         - If scene has textOverlay: create a Text element with style from preset
         - Add transitions between consecutive clips using style.transitionType
         - Return `{ elements: [...], transitions: [...], duration: totalDuration, settings: { width: 1920, height: 1080, fps: 30 } }`
       - Export interface `Scene { description: string; duration: number; mood?: string; textOverlay?: string }`
       - Export interface `ProjectComposition { elements: any[]; transitions: any[]; duration: number; settings: { width: number; height: number; fps: number } }`

    2. `editor/src/app/api/ai/text-to-video/route.ts` - API endpoint:
       - `POST /api/ai/text-to-video` - Generates video composition from scenes
       - Request body:
         ```
         {
           scenes: Array<{ description: string; duration: number; textOverlay?: string }>;
           styleId: string;  // from VIDEO_STYLE_PRESETS
         }
         ```
       - Validate: scenes array required and non-empty, each scene needs description and duration, styleId must match a known preset
       - Create `TextToVideoService` instance with `process.env.ANTHROPIC_API_KEY`
       - Call `service.generateFromStoryboard(scenes, styleId)`
       - Return `{ composition: ProjectComposition, scenesWithClips: Array<{ scene: Scene; clip: StockClip | null }> }` with 200
       - scenesWithClips provides the mapping so the UI can show which clip was selected per scene
       - Handle errors: 400 for validation, 500 for generation failures
       - Note: This endpoint can take 10-30 seconds due to LLM + stock API calls. Set appropriate timeout.
