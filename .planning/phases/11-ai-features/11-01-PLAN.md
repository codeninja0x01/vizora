  <action>
    Create the AI module foundation and TTS provider abstraction:

    1. `editor/src/lib/ai/types.ts` - Shared AI types:
       - Export type `AIProvider = 'elevenlabs' | 'openai'`
       - Export type `AIConfig` with optional API keys for each provider

    2. `editor/src/lib/ai/providers/tts/types.ts` - TTS provider interface:
       - `Voice` type: `{ id: string; name: string; previewUrl?: string; provider: AIProvider; language?: string; gender?: string }`
       - `TTSRequest` type: `{ text: string; voiceId: string; speed?: number; provider: AIProvider }`
       - `TTSResponse` type: `{ audioUrl: string; duration: number; format: 'mp3' | 'wav'; provider: string }`
       - `ITTSProvider` interface with methods: `listVoices(): Promise<Voice[]>`, `synthesize(request: TTSRequest): Promise<TTSResponse>`, `getProvider(): AIProvider`

    3. `editor/src/lib/ai/providers/tts/elevenlabs.ts` - ElevenLabs adapter:
       - Class `ElevenLabsProvider implements ITTSProvider`
       - Constructor takes `{ apiKey: string; baseUrl: string; model: string }` - read from env vars with defaults
       - `listVoices()`: Fetch `GET {baseUrl}/v1/voices`, map response to Voice[] with provider='elevenlabs'
       - `synthesize()`: Import and call `parsePauseMarkers(text)` to convert [pause Xs] to SSML `<break time="Xs"/>`, then POST to `{baseUrl}/v1/text-to-speech/{voiceId}` with model_id, voice_settings (stability: 0.5, similarity_boost: 0.75). Return audio as ArrayBuffer.
       - Port existing logic from `/api/elevenlabs/voiceover/route.ts` but structured as a class

    4. `editor/src/lib/ai/providers/tts/openai-tts.ts` - OpenAI TTS adapter:
       - Install `openai` package: `pnpm add openai --filter editor`
       - Class `OpenAITTSProvider implements ITTSProvider`
       - Constructor takes `{ apiKey: string }`
       - `listVoices()`: Return hardcoded list of OpenAI voices (alloy, ash, ballad, coral, echo, fable, nova, onyx, sage, shimmer) with provider='openai'
       - `synthesize()`: Use `openai.audio.speech.create({ model: 'gpt-4o-mini-tts', voice: voiceId, input: stripPauseMarkers(text), speed })`. Strip pause markers since OpenAI TTS doesn't support SSML breaks - use `stripPauseMarkers` utility. Convert response to ArrayBuffer.
       - Note: OpenAI TTS returns raw audio bytes, not a URL

    5. `editor/src/lib/ai/providers/tts/factory.ts` - Provider factory:
       - `TTSProviderFactory` class with static `create(provider: AIProvider): ITTSProvider`
       - Read API keys from `process.env.ELEVENLABS_API_KEY` and `process.env.OPENAI_API_KEY`
       - Throw clear error if requested provider's API key is not configured
       - Static `getAvailableProviders(): AIProvider[]` - returns providers whose API keys are present

    6. `editor/src/lib/ai/utils/pause-markers.ts` - Pause marker utilities:
       - `parsePauseMarkers(text: string): string` - Replaces `[pause 1s]` or `[pause 500ms]` with SSML `<break time="1s"/>`. Uses regex: `/\[pause\s+(\d+(?:\.\d+)?)(s|ms)\]/gi`
       - `stripPauseMarkers(text: string): string` - Removes all pause markers from text entirely (for providers that don't support SSML). Inserts a period and space at marker positions to create natural pauses in speech.
       - `extractPauseMarkers(text: string): Array<{ position: number; duration: number }>` - Returns pause positions and durations for timeline positioning
