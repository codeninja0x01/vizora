1. Create editor/src/app/api/v1/webhooks/[id]/route.ts:

All three handlers wrapped with withApiAuth. Extract webhook `id` from the URL path using `new URL(request.url).pathname.split('/').at(-1)` or the standard Next.js route params pattern.

**GET handler (get single webhook):**
- Find webhook by id AND organizationId (prevent cross-org access)
- Return 404 if not found
- Return webhook without secret: `{ id, url, enabled, lastDeliveryAt, lastSuccessAt, lastFailureAt, consecutiveFailures, createdAt, updatedAt }`

**PATCH handler (update webhook):**
- Parse body with Zod: `{ url: z.string().url().optional(), enabled: z.boolean().optional() }`
- If url provided, validate with `validateWebhookUrl()` — return 400 if invalid
- Find webhook by id AND organizationId — return 404 if not found
- Update only provided fields via prisma.webhookConfig.update
- Return updated webhook (without secret)

**DELETE handler (delete webhook):**
- Find webhook by id AND organizationId — return 404 if not found
- Delete via prisma.webhookConfig.delete
- Return 204 No Content (empty body)

Note: For route params in Next.js App Router, the handler signature should accept the second parameter with params: `async function handler(request: Request, context: ApiKeyContext)`. Since withApiAuth strips the native params, extract the ID from the URL path instead: `const id = new URL(request.url).pathname.split('/').filter(Boolean).at(-1)` — but be careful with the rotate-secret sub-route. For [id] route, the ID is the segment after "webhooks": parse from URL pathname matching `/api/v1/webhooks/([^/]+)$`.

2. Create editor/src/app/api/v1/webhooks/[id]/rotate-secret/route.ts:

- Wrap with withApiAuth
- Extract webhook ID from URL path (segment before "rotate-secret")

- Update secret in database via prisma.webhookConfig.update
- Reset consecutiveFailures to 0
- Return 200 with: `{ id, secret, rotatedAt: new Date().toISOString() }` — new secret shown once
- Include `"_note": "Store this secret securely. It will not be shown again."`
