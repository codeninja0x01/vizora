**New model: CreditTransaction** (after Batch model):
```prisma
model CreditTransaction {
  id             String   @id @default(cuid())
  organizationId String
  amount         Int      // Negative for deductions, positive for additions
  balanceBefore  Int
  balanceAfter   Int
  reason         String   // "render", "subscription_renewal", "credit_pack", "rollover", "refund_system_failure"
  renderId       String?  // Null for non-render transactions
  metadata       Json?    // Duration, pack size, rate used, etc.
  createdAt      DateTime @default(now())

  organization Organization @relation(fields: [organizationId], references: [id], onDelete: Cascade)

  @@index([organizationId, createdAt])
  @@index([reason])
}
```

**New model: SubscriptionEvent** (after CreditTransaction):
```prisma
model SubscriptionEvent {
  id              String   @id @default(cuid())
  organizationId  String
  stripeEventId   String   @unique // For idempotency
  eventType       String   // checkout.session.completed, invoice.paid, etc.
  processed       Boolean  @default(false)
  metadata        Json?
  createdAt       DateTime @default(now())

  organization Organization @relation(fields: [organizationId], references: [id], onDelete: Cascade)

  @@index([organizationId])
}
```

if (!process.env.STRIPE_SECRET_KEY) {
  console.warn('[Stripe] STRIPE_SECRET_KEY not configured - billing features disabled');

**Create `editor/src/lib/billing.ts`** -- Billing constants and credit calculation:

Export a `CREDIT_RATE_PER_SECOND` constant set to `10` (10 credits per second of video output, per research recommendation).

Export a `TIER_CONFIG` record mapping tier names to their configuration:
- `free`: monthlyAllotment 3000, concurrentLimit 5, displayName "Free", priceMonthly 0
- `pro`: monthlyAllotment 30000, concurrentLimit 50, displayName "Pro", priceMonthly 29 (dollars)
- `enterprise`: monthlyAllotment 1000000, concurrentLimit Infinity, displayName "Enterprise", priceMonthly null (contact sales)

Each tier config should include: `monthlyAllotment: number`, `concurrentLimit: number`, `displayName: string`, `priceMonthly: number | null`, `maxRollover: number` (2x monthlyAllotment per user decision).

Export credit pack definitions `CREDIT_PACKS` array:
- { id: 'pack_1000', credits: 1000, priceUsd: 10, label: '1,000 Credits' }
- { id: 'pack_5000', credits: 5000, priceUsd: 40, label: '5,000 Credits' }
- { id: 'pack_15000', credits: 15000, priceUsd: 100, label: '15,000 Credits' }

Export `calculateCredits(durationSeconds: number): number` -- returns `Math.ceil(durationSeconds * CREDIT_RATE_PER_SECOND)`. Use Math.ceil to round up (never charge 0 for a short video).

Export `LOW_CREDIT_THRESHOLD = 0.2` (20% per user decision).

Export `isLowCredit(balance: number, monthlyAllotment: number): boolean` -- returns `balance <= monthlyAllotment * LOW_CREDIT_THRESHOLD`.

Export `canRender(subscriptionStatus: string | null, tier: string): boolean` -- returns false if subscriptionStatus is 'past_due' or 'canceled' (unless tier is 'free' which has no subscription).

Export TypeScript types: `TierName = 'free' | 'pro' | 'enterprise'`, `TierConfig`, `CreditPack`.

Use proper TypeScript types throughout. No `any`.
