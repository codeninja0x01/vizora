**Create `editor/src/components/billing/low-credit-banner.tsx`:**

A `'use client'` component that shows a dismissable warning banner when credits are low.

Props interface:
```typescript
interface LowCreditBannerProps {
  creditBalance: number;
  monthlyAllotment: number;
  tier: string;
}
```

Implementation:
- Use `useState` for `dismissed` (default false)
- Calculate: `isLow = monthlyAllotment > 0 && creditBalance <= monthlyAllotment * 0.2`
- If not low, or dismissed, or tier is 'free' and monthlyAllotment is 0 (no subscription), return null
- Calculate percentage: `Math.round((creditBalance / monthlyAllotment) * 100)`
- Render a banner:
  - Full-width bar at top of content area: `bg-amber-500/10 border border-amber-500/20 rounded-lg px-4 py-3 mb-6`
  - If balance is 0: use red styling instead: `bg-red-500/10 border-red-500/20`
  - Left side: Warning icon (AlertTriangle from lucide-react) + text:
    - If balance > 0: "Your credits are running low ({percentage}% remaining). {creditBalance} credits left."
    - If balance === 0: "You're out of credits. Renders are blocked until you upgrade or buy credits."
  - Right side: Two elements:
    - Link to `/dashboard/billing`: "Upgrade Plan" or "Buy Credits" button (small, outlined style)
    - Dismiss button (X icon) that sets dismissed=true
  - Use `text-amber-400` for warning text, `text-red-400` for zero-credit text

**Modify `editor/src/app/(protected)/layout.tsx`:**

The layout is a server component. Add a query to fetch the organization's billing data for the banner.

1. After the existing session check and organization auto-creation, fetch billing data:
   ```typescript
   // Fetch billing data for low-credit banner
   const orgBilling = await prisma.organization.findUnique({
     where: { id: session.session.activeOrganizationId! },
     select: { creditBalance: true, monthlyAllotment: true, tier: true }
   });
   ```

2. Import LowCreditBanner component.

3. Add the banner inside the `<main>` element, before the `<div className="mx-auto max-w-6xl...">` wrapper:
   ```tsx
   <main className="flex-1 overflow-y-auto">
     <div className="mx-auto max-w-6xl px-6 py-8 lg:px-10 lg:py-10">
       {orgBilling && (
         <LowCreditBanner
           creditBalance={orgBilling.creditBalance}
           monthlyAllotment={orgBilling.monthlyAllotment}
           tier={orgBilling.tier}
         />
       )}
       {children}
     </div>
   </main>
   ```

Note: The banner dismissal is client-side only (resets on page reload). This is intentional -- it's a gentle reminder, not a blocking modal. The email notification (handled in Plan 03) is the persistent warning channel.

The org query is a single select with 3 lightweight fields on an already-indexed primary key -- negligible performance impact on every page load.
