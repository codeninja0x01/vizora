<task type="auto">
  <name>Task 1: Billing server actions</name>
  <files>editor/src/app/(protected)/dashboard/billing/actions.ts</files>
  <action>
Create `editor/src/app/(protected)/dashboard/billing/actions.ts` with `'use server'` directive.

Import: auth from `@/lib/auth`, prisma from `@/lib/db`, stripe from `@/lib/stripe`, TIER_CONFIG/CREDIT_PACKS from `@/lib/billing`, headers from `next/headers`, redirect from `next/navigation`.

**1. `getBillingOverview()`**
- Get session via `auth.api.getSession({ headers: await headers() })`
- Fetch Organization with billing fields: tier, creditBalance, monthlyAllotment, billingCycleStart, billingCycleEnd, subscriptionStatus, cancelAtPeriodEnd, stripeCustomerId, stripeSubscriptionId
- Calculate: creditPercentage (balance/allotment * 100), daysUntilRenewal, isLowCredit
- Return structured object with all display data plus TIER_CONFIG and CREDIT_PACKS

**2. `getUsageData()`**
- Get session, get organizationId
- Fetch recent CreditTransactions (last 30, ordered by createdAt desc):
  - Select: id, amount, balanceBefore, balanceAfter, reason, renderId, createdAt, metadata
- Fetch render stats: total renders this billing cycle grouped by status using prisma.render.groupBy
- Fetch total credits used this cycle: sum of negative CreditTransactions since billingCycleStart
- Return: { transactions, renderStats, totalCreditsUsed, billingCycleStart }

**3. `createCheckoutSession(priceId: string)`**
- Get session, get organization (with stripeCustomerId)
- Validate stripe is configured, return error object if not
- Create or retrieve Stripe customer:
  - If org.stripeCustomerId exists, use it
  - Otherwise: `stripe.customers.create({ metadata: { organizationId } })`, save to Organization
- Create checkout session:
  ```typescript
  const session = await stripe.checkout.sessions.create({
    customer: customerId,
    mode: 'subscription',
    line_items: [{ price: priceId, quantity: 1 }],
    success_url: `${process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000'}/dashboard/billing?success=true`,
    cancel_url: `${process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000'}/dashboard/billing`,
    metadata: { organizationId: org.id }
  });
  ```
- Redirect to session.url

**4. `createPortalSession()`**
- Get session, get organization stripeCustomerId
- If no stripeCustomerId, return error
- Create portal session:
  ```typescript
  const session = await stripe.billingPortal.sessions.create({
    customer: org.stripeCustomerId,
    return_url: `${process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000'}/dashboard/billing`
  });
  ```
- Redirect to session.url

**5. `createCreditPackCheckout(packId: string)`**
- Get session, get organization stripeCustomerId
- Find pack in CREDIT_PACKS by id, validate exists
- Create or retrieve Stripe customer (same pattern as subscription checkout)
- Create checkout session with `mode: 'payment'`:
  ```typescript
  const session = await stripe.checkout.sessions.create({
    customer: customerId,
    mode: 'payment',
    line_items: [{
      price_data: {
        currency: 'usd',
        product_data: { name: pack.label, description: 'One-time credit top-up for OpenVideo' },
        unit_amount: pack.priceUsd * 100 // cents
      },
      quantity: 1
    }],
    success_url: `${process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000'}/dashboard/billing?credits_added=true`,
    cancel_url: `${process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000'}/dashboard/billing`,
    metadata: { organizationId: org.id, packSize: String(pack.credits), type: 'credit_pack' }
  });
  ```
- Redirect to session.url

**6. `submitEnterpriseContact(formData: FormData)`**
- Extract: name, email, company, message from formData
- Validate fields (name required, email required and valid format, message required)
- Send email via Resend (optional pattern) to a sales contact address (use process.env.SALES_EMAIL or fallback to the RESEND_FROM_EMAIL)
- Return `{ success: true }` or `{ success: false, error: 'message' }`

All actions must check session authentication. Return structured error objects (not throw) for client-side handling. Use `redirect()` for Stripe URL redirects.

  <action>
Create `editor/src/app/(protected)/dashboard/billing/page.tsx` as a server component page.

Follow existing dashboard page patterns (see dashboard/page.tsx, dashboard/renders/page.tsx for styling conventions). Use Tailwind CSS with the project's design tokens (oklch colors, bg-card, border-border/40, text-foreground, text-muted-foreground, etc.).

**Page structure:**

1. **Header section:**
   - Title: "Billing & Usage" with subtitle showing current tier badge
   - If URL has `?success=true`, show a success toast/banner: "Subscription activated successfully!"
   - If URL has `?credits_added=true`, show: "Credits added to your account!"

2. **Plan Overview Card** (top section):
   - Current plan name and tier badge (Free=gray, Pro=primary/indigo, Enterprise=amber)
   - Credit balance with visual progress bar: `{balance} / {monthlyAllotment} credits`
   - Progress bar color: green when >50%, amber when 20-50%, red when <20%
   - Billing cycle dates: "Renews {billingCycleEnd formatted}" or "No active subscription" for free
   - If cancelAtPeriodEnd: show "Cancels at end of period" warning
   - If subscriptionStatus === 'past_due': show red banner "Payment failed - rendering suspended. Update your payment method."

3. **Plan Comparison Section** (if on free tier or for upgrade):
   - Three plan cards side by side (responsive: stack on mobile)
   - Free card: "Current Plan" badge if active, 3,000 credits/mo, 5 concurrent renders, $0
   - Pro card: "Recommended" badge, 30,000 credits/mo, 50 concurrent renders, $29/mo, "Upgrade to Pro" button that calls createCheckoutSession with STRIPE_PRO_PRICE_ID (read from env via server action)
   - Enterprise card: 1,000,000 credits/mo, unlimited concurrent, "Contact Sales" - this section contains a small inline form (name, email, company, message) that calls submitEnterpriseContact
   - If already on Pro: Pro card shows "Current Plan", Free card shows "Downgrade via Portal", Enterprise shows "Contact Sales"

4. **Manage Subscription Section** (if has active subscription):
   - "Manage Subscription" button that calls createPortalSession (redirects to Stripe)
   - Text: "Update payment method, view invoices, or cancel your subscription"

5. **Credit Packs Section:**
   - Heading: "Buy Additional Credits"
   - Subtext: "One-time credit top-ups. Credits never expire."
   - Three pack cards from CREDIT_PACKS: each showing credits amount, price, price-per-credit
   - Each card has a "Buy" button that calls createCreditPackCheckout(packId)

6. **Usage Analytics Section:**
   - Heading: "Usage This Cycle"
   - Stats row: Total credits used, Renders completed, Renders failed (from getUsageData)
   - Credit Transaction History table (last 30 transactions):
     - Columns: Date, Description (derived from reason: "Render" / "Subscription Renewal" / "Credit Pack" / "System Refund"), Amount (green for +, red for -), Balance After
     - Use a simple HTML table with Tailwind styling (no need for a table library)

**Client interactivity:**
- Use `'use client'` wrapper components for interactive parts (form submissions, button clicks)
- The page itself can be a server component that fetches data via server actions, then renders client components
- Or use the pattern: server page fetches data, passes to client component
- For the checkout/portal buttons, use form actions or startTransition with server actions
- Use `sonner` toast for success/error feedback from URL params

**Styling guidance:**

- Stat cards: `rounded-lg bg-white/[0.03] p-4`
- Transaction table: `text-sm` with `border-b border-border/20` rows
- Match the existing dashboard aesthetic seen in other pages
