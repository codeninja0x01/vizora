**Create `editor/src/components/render/progress-bar.tsx`** — Progress bar with percentage and elapsed time (per locked decision: "clean horizontal bar, no stage labels"):

Props interface:
```typescript
interface ProgressBarProps {
  progress: number;     // 0-100
  startedAt: Date;      // For elapsed time calculation
  className?: string;
}
```

Implementation:
- Display percentage text (e.g., "67%") on the left and elapsed time (e.g., "2m 14s") on the right, in a flex row with `justify-between`
- Below the text, render a horizontal bar: outer container `h-2 bg-muted rounded-full overflow-hidden`, inner fill `h-full bg-primary rounded-full transition-all duration-300 ease-out` with `style={{ width: \`${progress}%\` }}`
- Elapsed time calculation: use a `useEffect` with 1-second interval that recalculates elapsed time from `startedAt` to now. Format as:
  - `< 60s`: `"{seconds}s"`
  - `1-59 min`: `"{min}m {sec}s"`
  - `60+ min`: `"{hr}h {min}m"`
  Use `date-fns` `intervalToDuration` and custom formatter, OR simple arithmetic (the custom formatter from the research section is cleaner and avoids date-fns edge cases for durations). Prefer the simple arithmetic approach from the research.
- Clean up the interval in the useEffect return.
- Use `text-sm` for percentage, `text-xs text-muted-foreground` for elapsed time.

Create the main render display component that composes the three components from Task 1. Per locked decisions: "No dedicated render detail page - click to expand a render entry in-place" and "Expanded view shows full details: progress (if active), error messages (if failed), output video player (if complete), metadata".

**Define the render data type** (at top of file or in a shared types location):
```typescript
interface RenderCardData {
  id: string;
  status: 'queued' | 'active' | 'completed' | 'failed';
  templateName: string;
  templateThumbnail?: string | null;
  createdAt: string;        // ISO string
  startedAt?: string | null;
  completedAt?: string | null;
  failedAt?: string | null;
  outputUrl?: string | null;
  errorCategory?: string | null;
  errorMessage?: string | null;
  resolution?: string | null;
  fileSize?: number | null;
  // Live progress from SSE (passed as prop, updated by parent)
  liveProgress?: number | null;
}
```

**Props interface:**
```typescript
interface RenderCardProps {
  render: RenderCardData;
}
```

**Component implementation:**

Use `useState` for `isExpanded` (boolean, default false). This is a 'use client' component.

**Collapsed state (always visible):**
Layout: A clickable row with `onClick={() => setIsExpanded(!isExpanded)}`.

```
[ChevronRight/Down] [Thumbnail 80x48] [Name + Status + Timestamp]  [Duration/Size]
```

- Left: `ChevronRight` icon (from lucide-react) when collapsed, `ChevronDown` when expanded. Size: `size-4 text-muted-foreground`.
- Thumbnail: `w-20 h-12 object-cover rounded flex-shrink-0`. If no thumbnail, show Layers icon (matching template card pattern from Phase 3) on gradient background `bg-gradient-to-br from-primary/20 to-primary/5`.
- Name and metadata: Template name as `font-medium truncate text-sm text-foreground`. Below it: `RenderStatusBadge` + created timestamp formatted as relative time (e.g., "2h ago") using same pattern as template-card.tsx. Use `text-xs text-muted-foreground`.
- Right side (when completed): render duration (completedAt - startedAt) formatted as "Xm Ys" and file size formatted as "XX.X MB". Use `text-xs text-muted-foreground`.

**Active renders (always visible, not just when expanded):**
If `status === 'active'` and `liveProgress` is available, show the `ProgressBar` component below the collapsed header row. Pass `progress={liveProgress}` and `startedAt={new Date(render.startedAt)}`.

If `status === 'active'` and `liveProgress` is null/undefined, show `ProgressBar` with `progress={0}` (will update when SSE provides data).

**Expanded state (shown when isExpanded is true):**
Render below the header, separated by `pt-3 border-t border-border/50 mt-3`.

Content depends on status:

1. **Metadata grid** (all statuses): 2-column grid at `text-xs`:
   - Render ID: `render.id` (truncated with copy-on-click is nice-to-have, not required)
   - Resolution: `render.resolution || 'Default'`
   - Created: Full date/time from `render.createdAt`
   - Started: Full date/time from `render.startedAt` (if available)
   - Completed: Full date/time from `render.completedAt` (if available)
   - Duration: computed from startedAt to completedAt
   - File size: formatted (if available)
   Label style: `text-muted-foreground`, value style: `text-foreground`

2. **Completed renders**: Show `VideoPreview` component with `videoUrl={render.outputUrl}`, `thumbnailUrl={render.templateThumbnail}`, `fileName={\`${render.templateName}.mp4\`}`.

3. **Failed renders**: Show error box with `p-3 bg-red-500/5 border border-red-500/20 rounded-lg` containing:
   - Error category badge (if available): `text-xs font-medium text-red-400`
   - Error message: `text-sm text-red-300`

4. **Queued renders**: Show `text-sm text-muted-foreground` message "Waiting in queue..."

**Card container styling:** Follow existing dashboard card patterns:
- `border border-border rounded-lg p-4 bg-card/50 hover:bg-card/80 transition-colors`
- Use `cursor-pointer` on the header area
- When expanded: no extra border changes (expansion is visible via chevron direction)

**Helper functions** (inline or at bottom of file):
- `formatRelativeTime(date: string)` — Same pattern as template-card.tsx (Xm ago, Xh ago, Xd ago)
- `formatDuration(start: string, end: string)` — Returns "Xm Ys" format
- `formatFileSize(bytes: number)` — Returns "XX.X MB" or "XX KB"
