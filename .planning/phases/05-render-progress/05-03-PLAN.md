**Create `editor/src/hooks/use-render-events.ts`** — SSE client hook with reconnection:

'use client' hook that manages an EventSource connection to `/api/v1/renders/events`.

```typescript
interface RenderEvent {
  type: 'connected' | 'progress' | 'completed' | 'failed';
  renderId?: string;
  activeCount?: number;
  data?: {
    progress?: number;
    outputUrl?: string;
    errorCategory?: string;
    errorMessage?: string;
  };
}

interface UseRenderEventsReturn {
  isConnected: boolean;
  activeCount: number;
  events: RenderEvent[];  // Recent events buffer (last 50)
  lastEvent: RenderEvent | null;
}
```

Implementation:
- Maintain `eventSourceRef` (useRef), `isConnected` (useState), `activeCount` (useState), `lastEvent` (useState)
- In `useEffect`, create `new EventSource('/api/v1/renders/events')`
- `onopen`: Set `isConnected = true`, reset reconnect attempts
- `onmessage`: Parse `event.data` as JSON. Handle by type:
  - `connected`: Set `activeCount` from event data
  - `progress`: Update `lastEvent`, keep track of activeCount (increment if new render seen)
  - `completed`: Update `lastEvent`, decrement activeCount
  - `failed`: Update `lastEvent`, decrement activeCount
- `onerror`: Set `isConnected = false`, close EventSource, attempt reconnection with exponential backoff:
  - Delays: 1s, 2s, 4s, 8s, 16s, max 30s
  - Max 10 reconnect attempts, then stop (user must refresh)
  - Clear reconnect timeout on cleanup
- Cleanup in useEffect return: close EventSource, clear any pending reconnect timeout
- Accept an optional `onEvent` callback prop for external consumers to react to events without re-rendering

**Create `editor/src/hooks/use-render-filters.ts`** — nuqs URL state hook:

'use client' hook wrapping nuqs for type-safe URL filter state.

```typescript
import { useQueryStates, parseAsString } from 'nuqs';

export function useRenderFilters() {
  const [filters, setFilters] = useQueryStates({
    status: parseAsString.withDefault('all'),
    search: parseAsString.withDefault(''),
    cursor: parseAsString.withDefault(''),
  });

  return {
    status: filters.status,
    search: filters.search,
    cursor: filters.cursor,
    setStatus: (status: string) => setFilters({ status, cursor: '' }),  // Reset cursor on filter change
    setSearch: (search: string) => setFilters({ search, cursor: '' }),  // Reset cursor on search change
    setCursor: (cursor: string) => setFilters({ cursor }),
    clearFilters: () => setFilters({ status: 'all', search: '', cursor: '' }),
  };
}
```

Note: `setStatus` and `setSearch` both reset cursor to '' because changing filters should go back to page 1.

```typescript
export async function getRenders(filters?: {
  status?: string;
  search?: string;
  cursor?: string;
  limit?: number;
})
```

Implementation:
1. Get session via `auth.api.getSession({ headers: await headers() })`. Throw if no session.
2. Get `activeOrgId` from `session.session.activeOrganizationId`.
3. Build Prisma `where` clause:
   - Always: `organizationId: activeOrgId`
   - If `filters.status` and `filters.status !== 'all'`: Map UI status to DB status:
     - 'rendering' -> 'active' (UI uses "Rendering" but DB uses "active")
     - Others map directly: 'queued', 'completed', 'failed'
   - If `filters.search`: Use `OR` clause:
     - `{ id: { startsWith: filters.search } }` (for render ID search — uses `startsWith` for cursor-compatible ID search)
     - `{ template: { name: { contains: filters.search, mode: 'insensitive' } } }` (for template name search)
4. Query with cursor-based pagination:
   ```typescript
   const PAGE_SIZE = filters?.limit || 20;
   const renders = await prisma.render.findMany({
     where,
     include: {
       template: {
         select: { name: true, thumbnailUrl: true },
       },
     },
     orderBy: { queuedAt: 'desc' },
     take: PAGE_SIZE + 1,
     ...(filters?.cursor ? {
       cursor: { id: filters.cursor },
       skip: 1,  // Skip the cursor item itself
     } : {}),
   });
   ```
5. Determine `hasMore` and `nextCursor`:
   ```typescript
   const hasMore = renders.length > PAGE_SIZE;
   const items = hasMore ? renders.slice(0, PAGE_SIZE) : renders;
   const nextCursor = hasMore ? items[items.length - 1].id : null;
   ```
6. Map results to client-safe objects (serialize dates as ISO strings):
   ```typescript
   return {
     items: items.map((r) => ({
       id: r.id,
       status: r.status,
       templateName: r.template.name,
       templateThumbnail: r.template.thumbnailUrl,
       createdAt: r.queuedAt.toISOString(),
       startedAt: r.startedAt?.toISOString() ?? null,
       completedAt: r.completedAt?.toISOString() ?? null,
       failedAt: r.failedAt?.toISOString() ?? null,
       outputUrl: r.outputUrl,
       errorCategory: r.errorCategory,
       errorMessage: r.errorMessage,
       resolution: null, // Will be populated when available from render options
       fileSize: null,   // Will be populated when storage integration exists
     })),
     nextCursor,
     hasMore,
   };
   ```

```typescript
interface RenderListProps {

}
```

State:
- `renders`: Array of render data (initialized from `initialRenders.items`)
- `hasMore`: boolean (from `initialRenders.hasMore`)
- `nextCursor`: string | null (from `initialRenders.nextCursor`)
- `isLoadingMore`: boolean

Hooks:
- `useRenderFilters()` for filter state
- `useRenderEvents()` for SSE connection

**SSE event handling:** Use a `useEffect` that watches `lastEvent` from `useRenderEvents()`:
- When `lastEvent.type === 'progress'`: Find the render in `renders` array by `renderId`. If found, update its `liveProgress` field. If not found (new render created since page load), call `getRenders()` to refetch.
- When `lastEvent.type === 'completed'`: Find render by `renderId`, update its `status` to 'completed' and set `outputUrl` from event data.
- When `lastEvent.type === 'failed'`: Find render by `renderId`, update its `status` to 'failed' and set `errorMessage` from event data.
- Use immutable state updates (spread into new array per coding-style.md).

**Filter/search handling:** Use a `useEffect` that watches filter state changes (status, search). When filters change, call `getRenders({ status, search })` server action and replace `renders` state with new results. Show a subtle loading state (opacity-50 on the card grid) during refetch.

**Load More button:** When clicked:
- Set `isLoadingMore = true`
- Call `getRenders({ status, search, cursor: nextCursor })`
- Append new items to existing `renders` array
- Update `hasMore` and `nextCursor`
- Set `isLoadingMore = false`
