**Create `editor/src/lib/render-events.ts`** â€” QueueEvents singleton with user subscription management:

This module provides centralized BullMQ event monitoring and routes events to per-user SSE connections.

**Module-level state:**
- `queueEvents: QueueEvents | null` â€” singleton instance
- `userListeners: Map<string, Set<(event: RenderEvent) => void>>` â€” userId to callback sets
- `renderOwnerMap: Map<string, string>` â€” renderId to userId for event routing

**Types:**
```typescript
interface RenderEvent {
  type: 'progress' | 'completed' | 'failed';
  renderId: string;
  data: {
    progress?: number;
    outputUrl?: string;
    errorCategory?: string;
    errorMessage?: string;
    templateName?: string;
  };
}
```

**Exported functions:**

1. `initQueueEvents()` â€” Creates QueueEvents singleton (idempotent). Uses globalThis pattern (like db.ts) to survive HMR. Connects to Redis via redisConnection config (same host/port/password). Attaches three event listeners:

   - `'progress'` event: Parse `data` (which is the object from `job.updateProgress()`). Extract userId from `data.userId`. Update renderOwnerMap. Call all callbacks registered for that userId with `{ type: 'progress', renderId: jobId, data: { progress: data.percent } }`.

   - `'completed'` event: Parse `returnvalue` as JSON. Extract userId from parsed data. Call callbacks for userId with `{ type: 'completed', renderId: jobId, data: { outputUrl: parsed.outputUrl } }`. Clean up renderOwnerMap entry.

   - `'failed'` event: Look up userId from renderOwnerMap. Call callbacks for userId with `{ type: 'failed', renderId: jobId, data: { errorMessage: failedReason } }`. Clean up renderOwnerMap entry.

   Add error handler that logs `[QueueEvents] Error:` but does not crash.

2. `registerRender(renderId: string, userId: string)` â€” Pre-populate renderOwnerMap so failed events (which may not have prior progress events) can be routed correctly.

3. `subscribeUser(userId: string, callback: (event: RenderEvent) => void)` â€” Add callback to userListeners map for userId. Call `initQueueEvents()` to ensure singleton is running.

4. `unsubscribeUser(userId: string, callback: (event: RenderEvent) => void)` â€” Remove callback from userListeners. If set is empty, delete the userId entry.

**CRITICAL:** Use a separate Redis connection for QueueEvents (not the same instance as the queue). BullMQ docs require QueueEvents to have its own connection. Create connection config inline from the same env vars (REDIS_HOST, REDIS_PORT, REDIS_PASSWORD) with `maxRetriesPerRequest: null`.

**Create `editor/src/app/api/v1/renders/events/route.ts`** â€” Session-authenticated SSE endpoint:

```typescript
export const dynamic = 'force-dynamic';
export const runtime = 'nodejs'; // Required for streaming + Redis

export async function GET(request: Request) {
  // Authenticate via session cookie (EventSource sends cookies automatically)
  const session = await auth.api.getSession({
    headers: request.headers,
  });

  if (!session) {
    return new Response('Unauthorized', { status: 401 });
  }

  const userId = session.user.id;

  // Query user's currently active renders and register them
  const activeRenders = await prisma.render.findMany({
    where: {
      userId,
      status: { in: ['queued', 'active'] },
    },
    select: { id: true },
  });

  for (const render of activeRenders) {
    registerRender(render.id, userId);
  }

  // Create SSE stream
  const encoder = new TextEncoder();
  const stream = new ReadableStream({
    start(controller) {
      // Send initial connection event with active render count
      const initEvent = JSON.stringify({
        type: 'connected',
        activeCount: activeRenders.length,
      });
      controller.enqueue(encoder.encode(`data: ${initEvent}\n\n`));

      // Subscribe to render events for this user
      const callback = (event: RenderEvent) => {
        try {
          controller.enqueue(
            encoder.encode(`data: ${JSON.stringify(event)}\n\n`)
          );
        } catch {
          // Connection closed, will be cleaned up
        }
      };

      subscribeUser(userId, callback);

      // Heartbeat every 15 seconds to keep connection alive
      const heartbeat = setInterval(() => {
        try {
          controller.enqueue(encoder.encode(': heartbeat\n\n'));
        } catch {
          clearInterval(heartbeat);
        }
      }, 15000);

      // Cleanup on client disconnect
      request.signal.addEventListener('abort', () => {
        clearInterval(heartbeat);
        unsubscribeUser(userId, callback);
        try {
          controller.close();
        } catch {
          // Already closed
        }
      });
    },
  });

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache, no-transform',
      'Connection': 'keep-alive',
      'X-Accel-Buffering': 'no', // Disable nginx buffering
    },
  });
}
```

Import `auth` from `@/lib/auth`, `prisma` from `@/lib/db`, and `subscribeUser`, `unsubscribeUser`, `registerRender` from `@/lib/render-events`. Import the `RenderEvent` type too.
