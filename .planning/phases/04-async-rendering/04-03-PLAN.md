Output: Standalone worker process runnable via `npm run worker`, producing MP4 video files from template + merge data.

  <action>
Create `editor/src/workers/render-worker.ts` as a standalone entry point that runs independently from the Next.js application.

**Imports:**
- `Worker, Job` from 'bullmq'
- `Renderer` from '@designcombo/node' (workspace package - add as dependency in Task 2)
- `prisma` from relative path or via tsconfig alias to `@/lib/db`
- `redisConnection` from `@/lib/redis`
- `applyMergeData` from `@/lib/merge-fields`
- `categorizeError, formatRenderError` from `@/lib/error-categorization`
- `MergeField` type from `@/types/template`
- `mkdir` from 'fs/promises' for output directory
- `join` from 'path' for output path construction

**Constants:**
```typescript
const RENDER_OUTPUT_DIR = process.env.RENDER_OUTPUT_DIR || '/tmp/renders';
const RENDER_TIMEOUT_MS = 15 * 60 * 1000; // 15 minutes per user decision
```

**Output directory creation:**
```typescript
await mkdir(RENDER_OUTPUT_DIR, { recursive: true });
```

**Job processor function** `async (job: Job)`:

1. **Log job start** with structured format:
   `console.log(\`[Worker] Processing render ${job.data.renderId} (template: ${job.data.templateId})\`);`

2. **Update DB status to 'active':**
   ```typescript
   await prisma.render.update({
     where: { id: job.data.renderId },
     data: { status: 'active', startedAt: new Date() },
   });
   ```

3. **Fetch template with projectData** (full data, not the API-trimmed version):
   ```typescript
   const template = await prisma.template.findUnique({
     where: { id: job.data.templateId },
   });
   if (!template) {
     throw new Error(`Template ${job.data.templateId} not found`);
   }
   ```

4. **Apply merge data** using existing applyMergeData from merge-fields.ts:
   ```typescript
   const mergedProjectData = applyMergeData(
     template.projectData as Record<string, unknown>,
     template.mergeFields as MergeField[],
     job.data.mergeData || {},
   );
   ```

5. **Render video** using Renderer class from @designcombo/node:
   ```typescript
   const outputPath = join(RENDER_OUTPUT_DIR, `${job.data.renderId}.mp4`);
   const renderer = new Renderer({
     json: mergedProjectData,
     outputPath,
     browserOptions: {
       headless: true,
       timeout: RENDER_TIMEOUT_MS,
     },
   });

   // Log progress events
   renderer.on('progress', (progress) => {
     console.log(`[Worker] Render ${job.data.renderId}: ${progress.message}`);
   });

   await renderer.render();
   ```

6. **Update DB with completed status:**
   ```typescript
   const outputUrl = `/renders/${job.data.renderId}.mp4`;
   await prisma.render.update({
     where: { id: job.data.renderId },
     data: {
       status: 'completed',
       outputUrl,
       completedAt: new Date(),
     },
   });
   console.log(`[Worker] Render ${job.data.renderId} completed -> ${outputUrl}`);
   ```

7. **Return result** `{ outputPath, outputUrl }` for BullMQ's completed event.

**Error handling in try/catch around steps 3-6:**
```typescript
catch (error) {
  const { category, message } = formatRenderError(error);
  await prisma.render.update({
    where: { id: job.data.renderId },
    data: {
      status: 'failed',
      errorCategory: category,
      errorMessage: message,
      failedAt: new Date(),
    },
  });
  console.error(`[Worker] Render ${job.data.renderId} failed [${category}]: ${message}`);
  throw error; // Re-throw so BullMQ marks job as failed
}
```

**Worker configuration** (per user decisions):
```typescript
const worker = new Worker('render-queue', processor, {
  connection: redisConnection,
  concurrency: 1,              // 1 render at a time per worker
  lockDuration: 900000,         // 15 minutes (matches render timeout)
  stalledInterval: 60000,       // Check for stalled jobs every 1 minute
  maxStalledCount: 1,           // Fail job after 1 stall (no auto-retry)
});
```

**Event listeners:**
```typescript
worker.on('completed', (job) => {
  console.log(`[Worker] Job ${job.id} completed successfully`);
});

worker.on('failed', (job, error) => {
  console.error(`[Worker] Job ${job?.id} failed:`, error.message);
});

worker.on('error', (error) => {
  console.error('[Worker] Worker error:', error);
});

worker.on('stalled', (jobId) => {
  console.warn(`[Worker] Job ${jobId} stalled`);
});
```

**Graceful shutdown handlers:**
```typescript
async function shutdown(signal: string) {
  console.log(`[Worker] ${signal} received, shutting down gracefully...`);
  await worker.close();
  await prisma.$disconnect();
  console.log('[Worker] Worker shut down successfully');
  process.exit(0);
}

process.on('SIGTERM', () => shutdown('SIGTERM'));
process.on('SIGINT', () => shutdown('SIGINT'));
```

**Startup log:**
```typescript
console.log('[Worker] Render worker started, waiting for jobs...');
console.log(`[Worker] Output directory: ${RENDER_OUTPUT_DIR}`);
console.log(`[Worker] Timeout: ${RENDER_TIMEOUT_MS / 1000}s`);
```
