  <name>Task 1: Create POST /api/v1/renders submit endpoint with validation and queue limits</name>
  <files>
    editor/src/app/api/v1/renders/route.ts
  </files>
  <action>
Create `editor/src/app/api/v1/renders/route.ts` with POST and GET handlers, both wrapped with `withApiAuth` from `@/lib/api-middleware`.

**POST handler - Submit render:**

1. **Parse request body** - Extract `templateId` (string, required), `mergeData` (object, optional, default {}), `options` (object, optional) from JSON body. Validate with Zod:
   ```
   z.object({
     templateId: z.string().min(1),
     mergeData: z.record(z.unknown()).optional().default({}),
     options: z.object({
       width: z.number().int().positive().optional(),
       height: z.number().int().positive().optional(),
       fps: z.number().int().min(1).max(120).optional(),
       quality: z.number().min(0).max(1).optional(),
     }).optional(),
   })
   ```
   Return 400 with errors if body validation fails.

2. **Fetch template** - `prisma.template.findUnique({ where: { id: templateId } })` selecting id, mergeFields, mergeSchema, organizationId, isPublic. Return 404 if not found. Check access: template must be public OR owned by same organization as API key context.

3. **Validate merge data eagerly** (per user decision - instant 400 on invalid data, no wasted queue slot):
   - Use `validateMergeData()` from `@/lib/template-schema` with template's mergeFields (cast from Json) and the submitted mergeData
   - If validation fails, return 400 with `{ error: 'Validation failed', details: errors }` (NOT 422 - this is pre-queue validation, not the template validate endpoint)

4. **Enforce per-tier queue limits** (per user decision):
   ```
   const tierLimits: Record<string, number> = { free: 5, pro: 50, enterprise: Infinity };
   const maxQueued = tierLimits[context.tier] ?? 5;
   ```
   Count active+queued renders for the organization:
   `prisma.render.count({ where: { organizationId: context.organizationId, status: { in: ['queued', 'active'] } } })`
   If count >= maxQueued, return 429 with `{ error: 'Queue limit reached', message: 'Your {tier} plan allows up to {limit} concurrent render jobs.', limit, current }`

5. **Create render record in DB FIRST** (before queue add, so DB is always source of truth):
   ```
   const render = await prisma.render.create({
     data: {
       status: 'queued',
       templateId,
       userId: context.userId,
       organizationId: context.organizationId,
       mergeData: mergeData as any,
       queuedAt: new Date(),
     },
   });
   ```

6. **Add job to BullMQ queue** with render.id as the job ID:
   ```
   await renderQueue.add('render-video', {
     renderId: render.id,
     templateId,
     mergeData,
     options: options || {},
     userId: context.userId,
     organizationId: context.organizationId,
   }, { jobId: render.id });
   ```
   Use `render.id` as BullMQ jobId to correlate queue jobs with DB records.

7. **Return 202 Accepted**:
   ```json
   {
     "id": render.id,
     "status": "queued",
     "templateId": templateId,
     "createdAt": render.queuedAt.toISOString()
   }
   ```
   Headers: `Location: /api/v1/renders/${render.id}`, `Retry-After: 5`

**GET handler - List renders (in same file):**

Parse query parameters from URL:
- `status` (optional): filter by 'queued', 'active', 'completed', 'failed'
- `templateId` (optional): filter by template
- `fromDate` (optional): ISO date string, filter queuedAt >= date
- `toDate` (optional): ISO date string, filter queuedAt <= date
- `cursor` (optional, default '0'): offset for pagination
- `limit` (optional, default 20, max 100): page size

Build Prisma where clause scoped to `context.organizationId`. Apply optional filters.

Query with `prisma.render.findMany({ where, orderBy: { queuedAt: 'desc' }, skip: parseInt(cursor), take: limit + 1 })`.

Check if `results.length > limit` to determine `hasMore`. Slice to limit if needed. Calculate `nextCursor`.

Map results to response objects (exclude mergeData from list response for brevity):
```json
{
  "items": [{ "id", "status", "templateId", "queuedAt", "startedAt", "completedAt", "outputUrl" (if completed), "error" (if failed) }],
  "pagination": { "cursor": nextCursor, "hasMore", "limit" }
}
```

Return 200 with the response.

  <action>
Create `editor/src/app/api/v1/renders/[id]/route.ts` with GET handler wrapped with `withApiAuth`.

**GET handler - Poll render status:**

1. **Extract render ID** from URL path. In Next.js App Router with route params, use the second argument: `async (request: Request, context: ApiKeyContext)`. Since withApiAuth doesn't pass route params, extract ID from the URL:
   ```typescript
   const url = new URL(request.url);
   const segments = url.pathname.split('/');
   const renderId = segments[segments.length - 1]; // Last segment is the [id]
   ```

2. **Fetch render from database** (DB is source of truth per research):
   ```typescript
   const render = await prisma.render.findUnique({
     where: { id: renderId },
   });
   ```

3. **Check existence and ownership**:
   - If not found: return 404 `{ error: 'Not found', message: 'Render not found' }`
   - If `render.organizationId !== context.organizationId`: return 404 (don't leak existence)

4. **Return status-dependent response:**

   For `completed`:
   ```json
   {
     "id": render.id,
     "status": "completed",
     "templateId": render.templateId,
     "outputUrl": render.outputUrl,
     "queuedAt": render.queuedAt,
     "startedAt": render.startedAt,
     "completedAt": render.completedAt
   }
   ```
   Status: 200

   For `failed`:
   ```json
   {
     "id": render.id,
     "status": "failed",
     "templateId": render.templateId,
     "error": {
       "category": render.errorCategory,
       "message": render.errorMessage
     },
     "queuedAt": render.queuedAt,
     "startedAt": render.startedAt,
     "failedAt": render.failedAt
   }
   ```
   Status: 200

   For `queued` or `active`:
   ```json
   {
     "id": render.id,
     "status": render.status,
     "templateId": render.templateId,
     "queuedAt": render.queuedAt,
     "startedAt": render.startedAt
   }
   ```
   Status: 200, Headers: `Retry-After: 5` (suggest polling interval per user decision)

5. **Format all dates as ISO strings** using `.toISOString()`. Only include fields that have values (don't include null startedAt for queued status).
