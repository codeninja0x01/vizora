Create `editor/src/app/api/v1/assets/presigned/route.ts`:
- POST endpoint wrapped with `withApiAuth` (same pattern as /api/v1/renders)
- Request body: `{ filename: string, contentType: string, size: number, folderId?: string }`
- Validate with Zod: filename required, contentType must be in allowed MIME types, size <= 500MB
- Import `validateFileSize` from `@/lib/storage/validation.ts` for size check
- Generate R2 key: `assets/${context.organizationId}/${cuid()}/${sanitizedFilename}`
  - Use cuid() for unguessable UUID per user decision
  - Sanitize filename: replace spaces with hyphens, remove special chars
- Call `r2.createPresignedUpload(key, { contentType, expiresIn: 3600 })` to generate presigned URL
- Return `{ presignedUrl, r2Key, cdnUrl, contentType }` with 200 status
- The existing `/api/uploads/presign` endpoint is the old unauthenticated version — this new one replaces it for production use

Create `editor/src/app/api/v1/assets/route.ts`:
- POST endpoint (register asset after upload) wrapped with `withApiAuth`:
  - Body: `{ r2Key, name, contentType, size, category, folderId? }`
  - Validate with Zod
  - Create Asset record in DB with cdnUrl from r2.getUrl(r2Key)
  - Return 201 with asset data
- GET endpoint (list assets) wrapped with `withApiAuth`:
  - Query params: `folderId` (optional, null = root), `category` (optional), `cursor`, `limit`
  - Filter by `organizationId` from context
  - Return paginated assets with folder info
  - Include folder path breadcrumb data

Create `editor/src/app/api/v1/assets/[id]/route.ts`:
- GET endpoint: fetch single asset by ID, verify org ownership
- DELETE endpoint:
  1. Verify asset belongs to context.organizationId
  2. Check usage: search templates for asset CDN URL
     ```typescript
     const usageCount = await prisma.$queryRaw<[{ count: bigint }]>`
       SELECT COUNT(*) as count FROM "Template"
       WHERE "organizationId" = ${context.organizationId}
       AND CAST("projectData" AS TEXT) LIKE ${'%' + asset.cdnUrl + '%'}
     `;
     ```
  3. If usageCount > 0: return 409 Conflict with `{ error: "Asset is in use", references: Number(usageCount[0].count) }`
  4. If safe: delete from R2 via `r2.deleteObject(asset.r2Key)`, then delete DB record
  5. Return 204 No Content

IMPORTANT: Use `withApiAuth` for all endpoints — these are programmatic API endpoints. The editor UI will use server actions (Task 2) instead.

IMPORTANT: The usage check searches template projectData as text for the asset's CDN URL. This is a defensive check on delete (infrequent), so the text search performance is acceptable.

R2 key naming convention per Claude's discretion:
- Assets: `assets/{orgId}/{assetId}/{filename}` — org-scoped, UUID for unguessability
- Renders: `renders/{renderId}.mp4` — flat, render ID is already unique

Implement these server actions (follow Phase 3 server action patterns — auth.api.getSession, activeOrganizationId scoping):

1. `getFolders(parentId?: string)` — List folders for current org
   - If parentId is null, return root folders (parentId IS NULL)
   - Include `_count: { select: { assets: true, children: true } }` for counts
   - Order by name ascending

2. `createFolder(name: string, parentId?: string)` — Create new folder
   - Build materialized path from parent's path + name + "/"
   - If no parent: path = `/${name}/`
   - If parent exists: path = `${parent.path}${name}/`
   - Validate: name must be 1-50 chars, no slashes
   - Return created folder

3. `renameFolder(id: string, name: string)` — Rename folder
   - Update folder name and path (and all descendant paths)
   - For path update, use Prisma updateMany with startsWith on old path
   - This is the materialized path "move" operation from research

4. `deleteFolder(id: string)` — Delete folder
   - Check if folder has assets — if yes, return error (must move/delete assets first)
   - Delete folder (cascade deletes children per Prisma relation)

Create `editor/src/actions/asset-actions.ts`:
Server actions for asset management (editor UI):

1. `getAssets(params: { folderId?: string | null, category?: string, search?: string })` — List assets
   - Scope to current organization
   - Filter by folderId (null = root assets with no folder)
   - Optional category filter
   - Optional search by name (contains, case-insensitive)
   - Return with folder info, ordered by createdAt desc

2. `requestPresignedUrl(params: { filename: string, contentType: string, size: number, folderId?: string })` — Get presigned URL
   - Scope to current organization
   - Generate R2 key: `assets/${orgId}/${cuid()}/${sanitizedFilename}`
   - Call r2.createPresignedUpload with contentType
   - Return `{ presignedUrl, r2Key, cdnUrl }`

3. `registerAsset(params: { r2Key: string, name: string, contentType: string, size: number, category: AssetCategory, folderId?: string })` — Register uploaded asset
   - Create Asset record in DB
   - Return created asset
   - Revalidate editor path

4. `deleteAsset(id: string)` — Delete asset with usage check
   - Same usage check as API DELETE endpoint (search templates for CDN URL)
   - If in use: return `{ success: false, error: "Asset is used in N template(s). Remove from templates first." }`
   - If safe: delete from R2 + DB
   - Return `{ success: true }`
   - Revalidate editor path

5. `moveAssetToFolder(assetId: string, folderId: string | null)` — Move asset between folders
   - Update asset.folderId
   - Return updated asset

IMPORTANT: Server actions use `auth.api.getSession` from Better Auth for authentication (not API keys). The session provides userId and activeOrganizationId for scoping. Follow the exact pattern from Phase 3 template server actions.

IMPORTANT: The `requestPresignedUrl` action is the session-authenticated version for the editor UI. The /api/v1/assets/presigned route is the API-key-authenticated version for programmatic access. Both generate presigned URLs but use different auth mechanisms.
