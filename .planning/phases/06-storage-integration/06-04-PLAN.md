**Upload flow (react-dropzone):**
1. Use `useDropzone` hook with configuration:
   ```typescript
   const { getRootProps, getInputProps, isDragActive } = useDropzone({
     onDrop: handleFileDrop,
     maxSize: 500 * 1024 * 1024,  // 500 MB per user decision
     accept: {
       'video/*': ['.mp4', '.mov', '.webm'],
       'image/*': ['.png', '.jpg', '.jpeg', '.gif', '.svg'],
       'audio/*': ['.mp3', '.wav', '.aac'],
     },
     noClick: false,  // Allow click to open file picker
   });
   ```

2. `handleFileDrop` for each accepted file:
   a. Generate temp ID with `crypto.randomUUID()`
   b. Create object URL for immediate preview: `URL.createObjectURL(file)`
   c. Add to uploading map in store with status 'uploading', progress 0
   d. Validate file type via magic bytes:
      ```typescript
      const buffer = await file.slice(0, 4100).arrayBuffer();
      const { validateFileType } = await import('@/lib/storage/validation');
      const validation = await validateFileType(new Uint8Array(buffer), file.type);
      if (!validation.valid) {
        failUpload(tempId, validation.error);
        return;
      }
      ```
   e. Request presigned URL via server action: `requestPresignedUrl({ filename, contentType, size, folderId })`
   f. Upload directly to R2 with XMLHttpRequest for progress:
      ```typescript
      const xhr = new XMLHttpRequest();
      xhr.upload.addEventListener('progress', (event) => {
        if (event.lengthComputable) {
          updateUploadProgress(tempId, Math.round((event.loaded / event.total) * 100));
        }
      });
      xhr.open('PUT', presignedUrl);
      xhr.setRequestHeader('Content-Type', contentType);
      xhr.onload = () => { /* register asset */ };
      xhr.onerror = () => { failUpload(tempId, 'Upload failed'); };
      xhr.send(file);
      ```
   g. On upload success: call `registerAsset` server action
   h. On register success: replace uploading entry with real DB asset

3. **Inline progress display:**
   - Uploading assets shown at TOP of grid as special cards
   - Card shows: file preview (image/video thumbnail or audio icon), filename, progress bar
   - Progress bar: thin bar at bottom of card, indigo color, animated width
   - Per user decision: "Inline progress bar on asset thumbnail during upload"

4. **Asset grid (DB-backed):**
   - On mount: call `getAssets({ folderId: currentFolderId })` to fetch from DB
   - Display in grid: `grid-cols-[repeat(auto-fill,minmax(80px,1fr))]` (same as existing)
   - Reuse existing AssetCard pattern but with DB data
   - Click asset → add to canvas (same `addItemToCanvas` logic as existing code)
   - Delete button: call `deleteAsset` server action
     - If returns `{ success: false }` → show toast with error message (asset in use)
     - If returns `{ success: true }` → remove from grid

5. **Drag overlay state:**
   - When `isDragActive`: show overlay with dashed border and "Drop files here" text
   - Style: bg-primary/5, border-2 border-dashed border-primary/30, rounded-lg

6. **Load assets on mount and when currentFolderId changes:**
   ```typescript
   useEffect(() => {
     const loadAssets = async () => {
       setLoading(true);
       const assets = await getAssets({ folderId: currentFolderId, search: searchQuery, category: categoryFilter });
       setAssets(assets);
       setLoading(false);
     };
     loadAssets();
   }, [currentFolderId, searchQuery, categoryFilter]);
   ```

IMPORTANT: Remove all localStorage and OPFS persistence logic from the existing component. The DB is now the source of truth. The old `storageService`, `STORAGE_KEY`, `PROJECT_ID` references are no longer needed.

IMPORTANT: Keep the existing `addItemToCanvas` function logic — it already handles Image.fromUrl, Video.fromUrl, Audio.fromUrl for adding clips to the studio. Just update it to use the asset's cdnUrl.
